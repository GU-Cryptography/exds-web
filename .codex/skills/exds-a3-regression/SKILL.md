# exds-a3-regression

## 适用场景

1. 权限相关改动合入前。
2. 版本发布前。
3. 每周例行权限回归。

## 输入约定

1. 批次编号（如 `A3-20260321-01`）。
2. 执行环境（`dev/test/prod`）。
3. 代码版本（commit hash）。
4. 本次重点关注模块（可选）。

## 执行步骤（轻量固定流程）

1. 打开模板文件：`docs/pages/用户权限管理/A3-角色场景回归报告-轻量版.md`。
2. 按模板填写批次信息。
3. 执行固定最小必测集：
- 页面访问（R）
- 按钮可用（B）
- 写请求结果（Q/S）
4. 角色范围固定：
- `viewer`
- `analyst`
- `business_admin`
- `system_admin`
5. 记录矩阵结论，仅使用：
- `PASS`
- `FAIL`
- `N/A`
6. 如有问题，必须填写异常闭环：
- 现象
- 根因
- 修复
- 复测结论
7. 输出报告文件到同目录归档。

## 通过标准

1. 无越权成功。
2. 无误拦截（应有权限却被阻断）。
3. 无“填写后保存才报无权限”主流程问题。
