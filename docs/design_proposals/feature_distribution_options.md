# 特征分布面板 - 可视化优化方案

当前实现：**Tabs + 水平条形图**。
优点：清晰展示单类目详情。
缺点：无法一眼以此看到全局分布，需要切换 Tab。

以下提供 3 种替代方案供选择：

## 方案 A：矩形树图 (Treemap)
**形式**：使用嵌套的矩形展示所有 7 大类及其下的标签。矩形面积代表标签出现的频次（客户数）。
- **布局**：大矩形为“分类”（如生产班次），内部小矩形为“标签”（如连续生产）。
- **优点**：
    - **全局视野**：无需切换 Tab，一眼看尽所有标签的权重。
    - **空间利用率高**：自动填充空间，适合展示大量层级数据。
    - **视觉重点**：占比大的标签（主力特征）非常醒目。
- **缺点**：
    - 占比极小的标签可能太小而无法显示文字。

## 方案 B：旭日图 (Sunburst / Multi-level Donut)
**形式**：多层同心环图。内环为“分类”，外环为“标签”。
- **优点**：
    - **视觉吸引力强**：现代感强，适合大屏展示。
    - **层级清晰**：径向展示层级关系。
    - **交互性好**：鼠标悬停内环可高亮外环相关标签。
- **缺点**：
    - 标签文字呈放射状排列，阅读体验略逊于水平文字。
    - 空间利用率略低（四角有留白）。

## 方案 C：分类指标卡片组 (Category Stats Cards)
**形式**：网格布局的小卡片组。每个分类一个独立小卡片，卡片内展示 Top 5 标签的迷你列表或迷你柱图。
- **布局**：例如 2x4 或 3x3 网格。每个卡片包含：分类标题 + 标签列表（名称 | 占比条 | 数值）。
- **优点**：
    - **信息密度适中**：结构化强，阅读像报表一样轻松。
    - **数据精确**：适合需要看具体数值的场景。
    - **整齐划一**：界面非常工整。
- **缺点**：
    - 相比图表，视觉冲击力稍弱。

## 推荐建议

- 如果侧重**探索发现**（看哪里是重点）：推荐 **方案 A (树图)**。
- 如果侧重**数据报表**（看具体数）：推荐 **方案 C (卡片组)**。
- 如果侧重**大屏展示**（酷炫）：推荐 **方案 B (旭日图)**。
