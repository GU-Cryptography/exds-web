# 时段特征挖掘 (Timeslot Profiling) - 业务需求文档

## 一、模块定位与目标

### 1.1 核心目标

通过对48个时段 (或96个时间点) 的历史数据统计分析，为交易人员提供：
- **时段规律发现**：识别哪些时段具有稳定的价格模式
- **交易策略指导**：明确哪些时段应做多/做空日前市场
- **风险量化**：评估每个时段的价格波动性和预测可靠性

### 1.2 典型应用场景

**场景1：日前市场报价策略**
- 问题：明天第15时段 (07:00-07:15) 应该如何报价？
- 分析：查看第15时段的历史特征
  - 过去30天，80%的时候实时价格低于日前价格 (平均低15元/MWh)
  - **策略**：日前市场少报或不报，等实时市场低价购电
  - **风险**：20%概率判断错误，最大亏损30元/MWh

**场景2：风险时段识别**
- 问题：哪些时段的价格最不稳定？
- 分析：查看价格标准差排名
  - 第85时段 (21:00-21:15) 标准差最大 (σ=50元/MWh)
  - **策略**：谨慎交易该时段，或采用保守策略

**场景3：套利机会挖掘**
- 问题：哪些时段的日前-实时价差最稳定且显著？
- 分析：查看价差一致性评分
  - 第10时段：90%正价差，平均+20元/MWh，标准差5元/MWh
  - **策略**：系统性做空日前市场该时段

### 1.3 与"现货趋势分析"的区别

| 维度 | 现货趋势分析 | 时段特征挖掘 |
|------|-------------|-------------|
| **时间聚合** | 按日期聚合 (时间序列) | 按时段聚合 (横截面) |
| **关注问题** | 价格随时间的变化 | 每个时段的特性 |
| **输出** | 趋势曲线、周期性规律 | 时段排名、策略建议 |
| **典型用户** | 风险管理、策略分析师 | 交易员、报价员 |

## 二、核心业务需求

### 需求1：时段统计分析表格

**业务问题**：每个时段的平均价格、波动性、价差特征是什么？

**分析维度**：

#### 1.1 基础价格统计

对于每个时段 `i` (i = 1...48 或 1...96)，计算周期内 (如30天) 的：

| 指标 | 公式 | 业务含义 |
|------|------|---------|
| 平均价格_RT | `mean(price_rt[i])` | 该时段的实时市场平均价格 |
| 平均价格_DA | `mean(price_da[i])` | 该时段的日前市场平均价格 |
| 价格标准差_RT | `std(price_rt[i])` | 该时段的价格波动性 |
| 最高价_RT | `max(price_rt[i])` | 该时段出现过的最高价格 |
| 最低价_RT | `min(price_rt[i])` | 该时段出现过的最低价格 |
| 极差_RT | `max - min` | 该时段的价格振幅 |

#### 1.2 价差特征统计

| 指标 | 公式 | 业务含义 |
|------|------|---------|
| 平均价差 | `mean(price_rt[i] - price_da[i])` | 该时段的平均财务偏差 |
| 价差标准差 | `std(price_rt[i] - price_da[i])` | 价差的不确定性 |
| 正价差占比 | `count(spread > 0) / total_days` | 该时段实时价高于日前价的概率 |
| 负价差占比 | `count(spread < 0) / total_days` | 该时段实时价低于日前价的概率 |
| 最大正价差 | `max(spread)` | 最大亏损幅度 (如果做多日前) |
| 最大负价差 | `min(spread)` | 最大盈利幅度 (如果做多日前) |

#### 1.3 交易指导指标

| 指标 | 计算逻辑 | 业务含义 |
|------|---------|---------|
| **一致性评分** | `max(正价差占比, 负价差占比)` | 价差方向的稳定性 (0.5-1.0) |
| **推荐策略** | 基于一致性评分和平均价差 | "做多日前" / "做空日前" / "观望" |
| **置信度** | 基于一致性和样本量 | "高" / "中" / "低" |
| **风险等级** | 基于价差标准差 | "高风险" / "中风险" / "低风险" |

**推荐策略逻辑**：
```python
if 一致性评分 >= 0.70:  # 70%以上时间价差方向一致
    if 平均价差 > 10:  # 平均正价差 > 10元/MWh
        推荐策略 = "做空日前 (实时买入)"
        置信度 = "高" if 一致性评分 >= 0.80 else "中"
    elif 平均价差 < -10:
        推荐策略 = "做多日前 (日前买入)"
        置信度 = "高" if 一致性评分 >= 0.80 else "中"
    else:
        推荐策略 = "观望"
        置信度 = "低"
else:
    推荐策略 = "观望 (方向不稳定)"
    置信度 = "低"
```

**风险等级逻辑**：
```python
if 价差标准差 < 15:
    风险等级 = "低风险"
elif 价差标准差 < 30:
    风险等级 = "中风险"
else:
    风险等级 = "高风险"
```

**数据来源**：
- `day_ahead_spot_price`
- `real_time_spot_price`

**计算逻辑**：
```python
# 伪代码
for timeslot in range(1, 49):  # 或 1-97 for 96点
    rt_prices = []
    da_prices = []
    spreads = []

    for date in date_range:
        rt_price = get_timeslot_data(real_time_spot_price, date, timeslot)
        da_price = get_timeslot_data(day_ahead_spot_price, date, timeslot)

        rt_prices.append(rt_price)
        da_prices.append(da_price)
        spreads.append(rt_price - da_price)

    stats = {
        'timeslot': timeslot,
        'avg_price_rt': mean(rt_prices),
        'avg_price_da': mean(da_prices),
        'std_price_rt': std(rt_prices),
        'max_price_rt': max(rt_prices),
        'min_price_rt': min(rt_prices),
        'avg_spread': mean(spreads),
        'std_spread': std(spreads),
        'positive_spread_ratio': count(spreads > 0) / len(spreads),
        'negative_spread_ratio': count(spreads < 0) / len(spreads),
        'max_spread': max(spreads),
        'min_spread': min(spreads),
        'consistency_score': max(positive_ratio, negative_ratio),
        'recommended_strategy': get_strategy(stats),
        'confidence': get_confidence(stats),
        'risk_level': get_risk_level(stats)
    }
```

### 需求2：时段价差分布可视化

**业务问题**：每个时段的价差分布形态是什么？是否存在极端值？

**分析方法**：箱线图 (Box Plot)

**数据结构**：
- X轴: 时段编号 (1-48 或 1-96)
- Y轴: 价差 (元/MWh)
- 每个箱体: 该时段在周期内的价差分布 (Q1, 中位数, Q3, 须线, 离群值)

**业务价值**：
- **识别对称性**: 箱体是否以零轴对称 (预测是否有偏差)
- **识别离群值**: 哪些时段出现过极端价差
- **对比波动性**: 箱体高度反映价差稳定性

**交互设计**：
- Hover 显示五数概括和离群值列表
- 点击箱体高亮该时段在统计表格中的行
- 支持筛选显示: 仅显示高一致性时段、仅显示高风险时段

---

### 需求3：高确定性时段雷达图

**业务问题**：快速识别最值得交易的时段

**可视化方法**：雷达图 (Radar Chart)

**数据维度**：
- 一致性评分 (0-100%)
- 平均价差绝对值 (归一化)
- 样本量充足度 (归一化)
- 低风险性 (1 - 归一化后的标准差)

**筛选条件**：
- 仅展示一致性评分 >= 70% 的时段 (Top N)
- 最多展示10个时段 (避免雷达图过密)

**业务价值**：
- 一眼识别"又稳又赚"的时段
- 快速对比不同时段的综合评分

---

### 需求4：时段对比分析

**业务问题**：某两个时段的特征差异是什么？

**交互设计**：
- 在统计表格中勾选2个时段
- 弹出对比卡片，并排展示两个时段的所有指标
- 高亮显示差异显著的指标 (如一个是"做多"，一个是"做空")

**数据展示**：

| 指标 | 时段15 (07:00) | 时段85 (21:00) | 差异 |
|------|---------------|---------------|------|
| 平均价格_RT | 180元/MWh | 220元/MWh | ↑ 40元 |
| 价格标准差 | 12元 | 48元 | ↑ 高风险 |
| 平均价差 | -15元 | +20元 | 反向 |
| 正价差占比 | 20% | 85% | 反向 |
| 推荐策略 | 做多日前 | 做空日前 | 反向 |

## 四、数据准备与依赖

### 4.1 数据完整性要求

- **最小样本量**: 至少20天数据 (30天为佳)
- **数据质量**: 96点数据完整，无缺失值
- **时段映射**: 需要明确的96点 → 48时段 → 尖峰平谷的映射规则

### 4.2 时段聚合规则

**96点 → 48时段**:
```python
# 每30分钟一个时段
时段1 = 平均(点1, 点2)  # 00:15 + 00:30
时段2 = 平均(点3, 点4)  # 00:45 + 01:00
...
```

**48时段 → 尖峰平谷**:
- 根据 `tou_rules` 将48时段映射到4个时段类型
- 每个类型内的时段单独统计

### 4.3 性能优化建议

- 时段统计计算量大 (96点 × 90天 × 多指标)
- 建议后端预计算并缓存结果
- 日期范围变化时触发后台异步计算
- 前端显示加载进度条

## 六、风险与限制

### 6.1 统计陷阱

**风险1：样本量不足**
- 问题：7天数据可能导致误判 (偶然性高)
- 缓解：强制要求最小样本量 >= 20天，并显示警告

**风险2：非平稳性**
- 问题：市场结构变化导致历史规律失效
- 缓解：支持"滚动窗口"分析，对比近期vs历史

**风险3：过度拟合**
- 问题：历史100%一致不代表未来100%
- 缓解：在UI上强调"置信度"和"风险等级"

### 6.2 策略建议的免责

**重要提示**：
- 推荐策略仅基于历史统计，不构成投资建议
- 用户应结合实时市场情况和自身风险承受能力决策
- 建议在UI上添加免责声明

## 七、后续优化方向

### 7.1 高级分析 (P2)

1. **条件分析**
   - 按工作日/周末分组统计
   - 按季节 (夏/冬) 分组统计
   - 按负荷水平 (高/中/低) 分组统计

2. **动态阈值**

3. **机器学习增强**
   - 使用分类模型预测下一个时段的价差方向
   - 置信度基于模型概率输出

### 7.2 交易辅助功能 (P3)

1. **报价模拟**

2. **实时提醒**
   - 当前时段是否符合推荐策略
   - 如果不符合，提供调整建议

3. **策略导出**
   - 导出高确定性时段列表为Excel
   - 一键生成"日前报价策略表"

## 九、附录：术语表

| 术语 | 定义 |
|------|------|
| 时段粒度 | 数据聚合的时间单位: 96点(15分钟)/48时段(30分钟)/尖峰平谷(时段类型) |
| 一致性评分 | 价差方向的稳定性，= max(正价差占比, 负价差占比)，范围[0.5, 1.0] |
| 做多日前 | 在日前市场买入，实时市场不操作 (预期实时价更高) |
| 做空日前 | 在日前市场不买或少买，等实时市场买入 (预期实时价更低) |
| 正价差 | price_rt > price_da，实时价高于日前价 (做多日前亏损) |
| 负价差 | price_rt < price_da，实时价低于日前价 (做多日前盈利) |
| 箱线图 | Box Plot，展示数据分布的五数概括: 最小值/Q1/中位数/Q3/最大值 + 离群值 |
| 置信度 | 推荐策略的可信程度，基于一致性评分和样本量 |
| 风险等级 | 基于价差标准差的风险分类: 高/中/低 |
| 样本量 | 统计计算所使用的有效天数 |
