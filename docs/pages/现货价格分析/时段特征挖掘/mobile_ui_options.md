# 时段分析页面 - 移动端UI设计方案

针对移动端屏幕较小、触控操作的特点，提供以下设计方案。

---

## 方案一：卡片流式列表 (Card Stream) - **推荐** 🌟

此方案采用经典的"卡片列表"模式，将表格行转换为独立的卡片，适合快速浏览和筛选大量时段数据。

### 1. 布局结构
- **顶部**: 汇总面板 (保持现有设计，自适应宽度)。
- **中间**: 图表区域 (保持现有设计，支持横屏全屏)。
- **底部**: **数据列表 (取代表格)**。
  - 桌面端：显示 `Table`。
  - 移动端：显示 `Stack` + `Card`。

### 2. 卡片设计细节
每个时段作为一个独立的卡片 (`Card`)，包含：
- **标题行**:
  - 左侧: 时段标签 (如 `00:00 - 00:30`) + 时段编号 (`#1`)。
  - 右侧: 推荐策略 (`Chip`) (如 "做多日前")。
- **核心指标区 (2x2 网格)**:
  - 左上: 平均价差 (带颜色: 红/绿)。
  - 右上: 一致性评分 (进度条)。
  - 左下: 风险等级 (Chip)。
  - 右下: 推荐指数 (信号强度条)。
- **视觉强化**:
  - 卡片左侧边框颜色根据平均价差正负变化 (红色=正价差, 绿色=负价差)。

### 3. 交互与功能
-   **排序功能**:
    -   在图表下方、卡片列表上方增加一个**排序工具栏**。
    -   工具栏包含一个下拉选择框，用于选择排序依据（例如：时段顺序、推荐指数、平均价差、一致性评分、波动性）。
    -   一个图标按钮用于切换升序/降序。
-   **加载更多**:
    -   卡片列表默认显示 **10** 条数据。
    -   当滚动到底部或点击"加载更多"按钮时，加载接下来的 **10** 条数据。
    -   当排序条件变化时，列表重置为默认显示 10 条。

### 4. 优缺点
- ✅ **优点**: 信息展示完整，无需横向滚动；符合用户在移动端刷"Feed流"的习惯；易于扩展更多指标；通过排序和加载更多功能提升了数据探索的灵活性。
- ❌ **缺点**: 页面整体长度较长，需要滚动浏览。

## 通用优化 (适用于移动端设计)

### 图表 X 轴优化
- **问题**: 在移动端竖屏状态下，图表 X 轴（时段编号）标签密集，容易重叠。
- **解决方案**:
    -   在移动端竖屏非全屏模式下，`XAxis` 的 `tickFormatter` 将调整为**每隔 6 个时段**显示一个标签。
    -   全屏或桌面端不受影响。
    -   用户仍可通过图表的 `Tooltip` 查看每个时段的详细信息。
