# 现货趋势分析 (Spot Market Trend Analysis) - 业务需求文档

## 一、模块定位与目标

### 1.1 核心目标

通过对现货市场历史数据的时间序列分析，帮助交易人员：
- **发现规律**：识别价格、价差的长期趋势和周期性规律
- **评估风险**：量化价格波动性和市场不确定性
- **辅助决策**：基于历史模式优化未来交易策略

### 1.2 与现有模块的关系

| 模块 | 关注点 | 时间尺度 |
|------|--------|---------|
| 市场价格总览 | 单日快照 | 1天 |
| 价差归因分析 | 单日偏差原因 | 1天 |
| **现货趋势分析** | **长期规律与模式** | **7-90天** |

### 1.3 目标用户

- **交易员**：寻找可套利的时段规律
- **风险管理**：评估市场波动性趋势
- **策略分析师**：制定中长期采购策略

## 二、核心业务需求

### 需求1：价格趋势分析

**业务问题**：近期现货价格是上涨还是下降？日前预测的准确性如何？

**分析维度**：
1. **日均价格趋势**
   - 每天的加权平均价 (VWAP): 反映真实购电成本
   - 每天的算术平均价 (TWAP): 反映平均价格信号
   - 分别展示日前 (DA) 和实时 (RT) 两个市场

2. **日均价差趋势**
   - VWAP价差 = VWAP_RT - VWAP_DA (财务偏差)
   - TWAP价差 = TWAP_RT - TWAP_DA (信号偏差)

3. **分时段价格趋势**
   - 按尖峰平谷分组，展示每个时段的日均VWAP趋势
   - 用途：识别特定时段的成本变化

**关键指标**：
- 日VWAP_RT、日VWAP_DA
- 日TWAP_RT、日TWAP_DA
- 日VWAP价差、日TWAP价差
- 尖峰平谷各时段的日均VWAP

**数据来源**：
- `day_ahead_spot_price`: 日前价格和电量
- `real_time_spot_price`: 实时价格和电量
- `tou_rules`: 分时电价规则 (用于时段分组)

**计算逻辑**：
```
每天的VWAP = Σ(price[i] × volume[i]) / Σ(volume[i])  (i = 1...96)
每天的TWAP = Σ(price[i]) / 96
时段VWAP = Σ(price[i] × volume[i]) / Σ(volume[i])  (i ∈ 该时段)
```

### 需求2：星期特性分析

**业务问题**：工作日和周末的价格模式有何不同？哪些星期更容易出现高价？

**分析维度**：
1. **周内价格分布**
   - 按星期一到星期日分组
   - 使用箱线图展示每个星期的VWAP/TWAP分布
   - 识别离群值 (异常高价/低价日)

2. **星期×时段热力图**
   - X轴: 96个时间点 (或聚合为24小时)
   - Y轴: 星期一到星期日
   - 颜色: 实时价格 (RT) 或价差 (RT-DA)
   - 用途: 可视化"周三下午容易高价"等模式

3. **周内价差模式**
   - 按星期分组，统计VWAP价差的平均值和标准差
   - 识别哪些星期的预测偏差更大

**关键指标**：
- 按星期分组的VWAP中位数、均值、25/75分位数
- 星期×时段的平均价格/价差矩阵
- 周内价差均值和波动性

**数据来源**：
- `day_ahead_spot_price`
- `real_time_spot_price`

**计算逻辑**：
```python
# 伪代码
for each_date in date_range:
    weekday = get_weekday(date)  # 0=周一, 6=周日
    daily_vwap = calculate_vwap(date)

# 箱线图数据
for weekday in [0, 1, 2, 3, 4, 5, 6]:

        'q3': percentile_75(weekday_data[weekday]),
        'min': min(weekday_data[weekday]),
        'max': max(weekday_data[weekday])
    }
```

### 需求3：波动性分析

**业务问题**：市场波动性是增加还是减少？哪些日期出现了剧烈波动？

**分析维度**：
1. **日内波动率**
   - 变异系数 (CV) = std(price_rt_96) / mean(price_rt_96)
   - 每天一个值，绘制时序曲线
   - 用途: 评估市场稳定性

2. **最大价格爬坡**
   - 每天的最大单点价格变化 = max(|price[i] - price[i-1]|)
   - 用途: 识别价格跳涨/跳水风险

3. **价差波动率**
   - std(price_spread_96)
   - 用途: 评估预测偏差的不确定性

**关键指标**：
- 日内价格变异系数 (CV)
- 日最大价格爬坡
- 日价差标准差

**数据来源**：
- `real_time_spot_price`
- `day_ahead_spot_price`

### 需求4：储能套利机会分析 (简化版)

**业务问题**：每天有哪些简单的低买高卖机会？价差有多大？

**分析维度**：

#### 5.1 分时段最大价差识别

将一天分为两个大时段，在每个时段内寻找最大的"低买高卖"价差：

**时段划分**：
- **上午时段**：00:00 - 11:30 (时间点 1-46)
- **下午时段**：11:30 - 23:30 (时间点 46-94，不含24:00即时间点96)

**算法逻辑**：
```python
# 对于每一天的96点价格
for 上午时段 (00:00 - 11:30, 时间点 1-46):
    找到最低价格点 price_min_am 和最高价格点 price_max_am
    约束: 最低价格点必须在最高价格点之前 (先买后卖)
    上午最大价差 = price_max_am - price_min_am
    记录: 最低价格、最高价格、时间点、价差

for 下午时段 (11:30 - 23:30, 时间点 46-94):
    找到最低价格点 price_min_pm 和最高价格点 price_max_pm
    约束: 最低价格点必须在最高价格点之前
    下午最大价差 = price_max_pm - price_min_pm
    记录: 最低价格、最高价格、时间点、价差

# 全天最大价差：在整个一天（96点）中寻找满足"先低后高"约束的最大价差
for 全天时段 (00:00 - 23:30, 时间点 1-94):
    找到最低价格点 price_min_all 和最高价格点 price_max_all
    约束: 最低价格点必须在最高价格点之前
    全天最大价差 = price_max_all - price_min_all
    记录: 最低价格、最高价格、时间点
```

**说明**：
- 上午价差和下午价差：分别在各自时段内寻找局部最优
- 全天最大价差：在整个一天寻找全局最优（通常 ≥ max(上午价差, 下午价差)）

**业务含义**：
- 上午价差：代表"凌晨低价充电 → 上午高价放电"的套利空间
- 下午价差：代表"中午低价充电 → 晚高峰高价放电"的套利空间
- 价差越大，储能套利机会越好

#### 5.2 周期价差趋势分析

对于分析周期内的每一天，计算以下指标并绘制时序曲线：

| 字段 | 说明 | 计算方法 |
|------|------|---------|
| 日期 | YYYY-MM-DD | - |
| 上午最低价 | 元/MWh | min(price[1:46]) |
| 上午最高价 | 元/MWh | max(price[最低价之后:46]) |
| 上午价差 | 元/MWh | 上午最高价 - 上午最低价 |
| 上午最低价时点 | HH:MM | 对应时间标签 |
| 上午最高价时点 | HH:MM | 对应时间标签 |
| 下午最低价 | 元/MWh | min(price[46:94]) |
| 下午最高价 | 元/MWh | max(price[最低价之后:94]) |
| 下午价差 | 元/MWh | 下午最高价 - 下午最低价 |
| 下午最低价时点 | HH:MM | 对应时间标签 |
| 下午最高价时点 | HH:MM | 对应时间标签 |
| 全天最大价差 | 元/MWh | 全天最高价 - 全天最低价 (满足先低后高约束) |
| 全天最低价 | 元/MWh | 全天96点中的最低价 (必须在最高价之前) |
| 全天最低价时点 | HH:MM | 对应时间标签 |
| 全天最高价 | 元/MWh | 全天96点中的最高价 (必须在最低价之后) |
| 全天最高价时点 | HH:MM | 对应时间标签 |
| 最优时段 | "上午" 或 "下午" | 哪个时段价差更大 |

#### 5.3 聚合统计指标

| 指标 | 公式 | 业务含义 |
|------|------|---------|
| 上午平均价差 | mean(上午价差) | 上午时段的平均套利空间 |
| 下午平均价差 | mean(下午价差) | 下午时段的平均套利空间 |
| 全天平均最大价差 | mean(全天最大价差) | 每天的平均最优套利空间 |
| 上午价差优势天数 | count(上午价差 > 下午价差) | 多少天上午更优 |
| 下午价差优势天数 | count(下午价差 > 上午价差) | 多少天下午更优 |
| 价差标准差 | std(全天最大价差) | 套利机会的稳定性 |
| 最大单日价差 | max(全天最大价差) | 最佳套利日 |
| 最小单日价差 | min(全天最大价差) | 最差套利日 |

**关键指标**：
- 上午/下午价差时序曲线
- 全天最大价差趋势
- 上午 vs 下午价差对比
- 价差分布箱线图

**数据来源**：
- `real_time_spot_price`: 实时价格96点
- `day_ahead_spot_price`: 日前价格96点 (可选，用于对比)

**计算复杂度**：
- 每天: O(n) = O(96) 线性扫描 ✅ 非常快
- 30天: 30 × 96 ≈ 2,880次计算 ✅ 毫秒级

**应用场景**：
- 储能运营商快速评估套利机会
- 对比日前预测价差 vs 实时价差，评估预测准确性
- 识别价差趋势变化，判断市场环境

#### 5.2 极值时段分布分析

**问题**：最高价和最低价更容易出现在哪些时段？

**分析方法**：
```python
# 统计周期内所有日期的极值时段分布
for each_date in date_range:
    max_price_timeslot = find_max_price_timeslot(date)
    min_price_timeslot = find_min_price_timeslot(date)

    # 统计96个时段的极值出现频次
    max_price_freq[max_price_timeslot] += 1
    min_price_freq[min_price_timeslot] += 1

# 按尖峰平谷分组统计
for period in ['尖峰', '峰', '平', '谷']:
    max_in_period = sum(max_price_freq[t] for t in period_timeslots)
    min_in_period = sum(min_price_freq[t] for t in period_timeslots)
```

**输出指标**：

| 时段类型 | 最高价出现次数 | 最高价占比 | 最低价出现次数 | 最低价占比 |
|---------|--------------|----------|--------------|----------|
| 尖峰 | 18 | 60% | 0 | 0% |
| 峰 | 8 | 27% | 2 | 7% |
| 平 | 4 | 13% | 5 | 17% |
| 谷 | 0 | 0% | 23 | 76% |

**业务含义**：
- 最高价主要出现在尖峰时段 (60%) → 风险集中
- 最低价主要出现在谷时段 (76%) → 低价购电窗口

#### 5.3 特殊价格事件识别

**定义特殊价格事件**：

| **超低价** | 0 < price < 50 | ℹ️ 低 | 极低价购电机会 |
| **超高价** | price > 1000 | 🔴 高 | 高成本风险，减少负荷 |
| **极端高价** | price > 2000 | 🔴 严重 | 极端风险，紧急应对 |

#### 6.5 异常价格预警规则

基于历史数据，设置预警阈值：

```python
# 动态阈值计算
historical_mean = mean(prices_in_period)
historical_std = std(prices_in_period)

# 预警阈值
warning_thresholds = {
    'high_price_warning': historical_mean + 2 * historical_std,  # 2σ预警
    'extreme_price_warning': historical_mean + 3 * historical_std,  # 3σ预警
    'low_price_opportunity': historical_mean - 2 * historical_std,  # 低价机会
}
```

**关键指标**：
- 每日最高价/最低价时序曲线
- 日极差时序曲线
- 极值时段分布 (尖峰平谷占比)
- 特殊价格事件统计 (负电价、零电价、超高价)
- 异常价格时段分布热力图
- 高风险时段列表

**数据来源**：
- `real_time_spot_price`: 实时价格96点
- `day_ahead_spot_price`: 日前价格96点 (可选，用于对比预测准确性)
- `tou_rules`: 分时电价规则 (用于时段分类)

**计算复杂度**：
- 每天: O(n) = O(96) 线性扫描 + O(1) 统计
- 30天: 非常快速 ✅

**应用场景**：
- **风险管理**: 识别高价风险时段，提前调整负荷

- **历史复盘**: 分析极端价格事件的成因和影响

## 六、后续优化方向

1. **预测性分析**: 基于历史趋势预测未来7天价格区间
2. **异常检测**: 自动标注异常高价/低价日期

## 七、开发优先级

### P0 (第一阶段 - 核心功能)
- 价格趋势分析 (需求1)
- 储能套利机会分析 (需求5)
- **价格异常与极值分析** (需求6) ⭐ **新增**
- 星期特性分析-箱线图 (需求2的核心)

### P1 (第二阶段 - 增强功能)
- 多因子相关性-负荷/火电/新能源 (需求3的前3项)
- 波动性分析 (需求4)

### P2 (第三阶段 - 高级分析)
- 星期×时段热力图 (需求2增强)
- 价格-温度相关性 (需求3的天气部分)
- 散点图矩阵 (需求3增强)

**优先级说明**：
- **需求5 (储能套利机会)** 提升至P0，因为：
  - 算法简单 (线性扫描 O(n))，实现快速
  - 业务价值明确 (直接指导储能充放电)
  - 数据需求最小 (仅需96点价格)
  - 可与需求1同步开发

- **需求6 (价格异常与极值)** 提升至P0，因为：
  - 算法极简单 (max/min查找 O(n))，实现快速
  - 风险管理核心需求 (识别超高价风险)
  - 机会识别重要工具 (发现零电价/负电价)
  - 可与需求1、5同步开发，共享数据处理逻辑

## 八、附录：术语表

| 术语 | 定义 |
|------|------|
| VWAP | Volume-Weighted Average Price，加权平均价，反映真实购电成本 |
| TWAP | Time-Weighted Average Price，算术平均价，反映平均价格信号 |
| RT | Real-Time，实时市场 |
| DA | Day-Ahead，日前市场 |
| CV | Coefficient of Variation，变异系数，std/mean |
| 皮尔逊相关系数 | 衡量两个变量线性相关程度，范围 [-1, 1] |
| 箱线图 | Box Plot，展示数据分布的五数概括 (最小值/Q1/中位数/Q3/最大值) |
| 热力图 | Heatmap，用颜色编码数值矩阵的可视化 |
