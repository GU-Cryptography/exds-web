# 现货价格分析模块 - 需求优化分析报告

> 生成时间: 2025-11-23
> 分析依据: 《江西市场现货价格特征及策略逻辑》业务研究文档

---

## 一、业务背景与核心目标

### 1.1 交易规则概述

本省电力交易采用**中长期交易 + 现货交易**的混合模式：

| 交易类型 | 电量占比 | 交易方式 | 结算方式 |
|---------|---------|---------|---------|
| 中长期交易 | 80% | 提前锁定价格和电量 | 按合同价结算 |
| 现货交易 | 20% | 日前交易（报量不报价） | 按实时价格结算 |

### 1.2 日前交易套利逻辑

**核心机制**：日前交易报量不报价，实际用电按实时价格结算，可通过预判日前-实时价差进行套利。

| 价差判断 | 操作策略 | 套利逻辑 | 操作名称 |
|---------|---------|---------|---------|
| 日前价格 > 实时价格（正差） | 卖出电量（含中长期持仓） | 高价卖出 → 低价结算 | **做空** |
| 日前价格 < 实时价格（负差） | 买入电量 | 低价买入 → 高价结算 | **做多** |

**示例**：
- 某时段日前价格 500元/MWh，预判实时价格 420元/MWh（正差+80元）
- 操作：做空，卖出该时段电量（包括中长期购买的电量）
- 结算：按420元/MWh实时价格结算实际用电
- 收益：每MWh套利80元

### 1.3 模块核心目标

**现货价格分析模块的根本目的**：

> 通过统计分析手段，实现对各时段**日前-实时价差方向和概率**的判断，从而指导操作员进行日前交易决策。

**核心输出**：
- 各时段正差/负差的历史概率
- 高确定性时段识别（概率>75%或<25%）
- 操作建议（做多/做空/观望）及置信度

## 二、核心发现

对比《江西市场现货价格特征及策略逻辑》（业务研究文档）与现有系统需求文档，发现以下问题：

**核心问题**：现有需求偏向"技术指标展示"，缺乏"日前交易决策支持"

具体表现：
1. 没有直接输出"各时段正负价差概率"
2. 没有区分工作日/周末/节假日的统计
3. 没有明确的"做多/做空/观望"操作建议
4. 缺乏高确定性时段的识别和置信度评估

---

## 三、缺失的核心分析功能

### 1. 时段价差概率统计 ⭐⭐⭐ 最核心

**业务需求**：
> 操作员需要知道"这个时段历史上正差概率是多少？应该做多还是做空？"

**系统现状**：
- "时段特征挖掘"有正价差占比，但没有分日类型统计
- 没有操作建议和置信度输出
- 没有月度/年度汇总视图

**建议新增**：
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分时段正负价差概率 | 各时段正差/负差出现的历史概率 | P0 |
| 分日类型统计 | 工作日/周末/节假日分别统计 | P0 |
| 操作建议输出 | 做多/做空/观望 + 置信度 | P0 |
| 月度/年度汇总 | 高确定性时段识别和规律总结 | P0 |

---

### 2. 竞价空间分析 ⭐⭐⭐

**业务文档洞察**：
> "10000–12000 MWh 是关键敏感带，5月价格对竞价空间更敏感"
> "当竞价空间 < 11000MW 时，可在中长期窗口减少持仓"

**对日前交易的影响**：
- 竞价空间小 → 价格波动大 → 价差不确定性高 → 谨慎操作
- 竞价空间大 → 价格稳定 → 价差可预测性高 → 可积极操作

**系统现状**：完全没有竞价空间相关的分析功能

**建议新增**：
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 竞价空间与价格关系图 | 散点图：X轴=竞价空间，Y轴=价格 | P0 |
| 竞价空间敏感区间识别 | 自动识别价格对空间变化最敏感的区间 | P0 |
| 竞价空间阈值预警 | 当竞价空间 < 阈值时提示风险 | P1 |

**数据来源**：需要新增 `竞价空间` 字段（= 系统负荷 - 新能源出力 - 非市场化出力）

---

### 3. 边界条件与价差关系分析 ⭐⭐⭐

**业务文档洞察**：
> "凌晨电价与风电出力呈明显反向关系"
> "白天价差更多由光伏预测偏差决定，夜间由负荷预测偏差主导"

**对日前交易的指导**：
- 白天：关注光伏预测偏差 → 光伏超预期 → 负差概率高 → 做多
- 傍晚：关注负荷预测偏差 → 负荷超预期 → 正差概率高 → 做空

**系统现状**：
- 有"价差归因分析"模块，但仅展示偏差值
- **没有量化边界条件对价差的影响机制**

**建议新增/增强**：
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 边界条件相关性矩阵 | 系统负荷/风电/光伏 与 价差的相关系数 | P0 |
| 分时段因子敏感性 | 识别"凌晨受风电影响"、"白天受光伏影响"等规律 | P0 |
| 预测偏差与价差散点图 | X轴=负荷/光伏偏差，Y轴=价差，分时段展示 | P1 |

---

### 4. 气象条件与价差关系 ⭐⭐

**业务文档洞察**：
> "地表短波辐射与价差几乎同步变化"
> "12–15时高辐射+中高风速+高温且无降水时是大正差高发期"
> "降水是突发型抑制因子"

**对日前交易的指导**：
- 高辐射+高风速+无降水 → 正差概率高 → 做空
- 降水天气 → 正差被抑制 → 观望或做多

**系统现状**：
- 有 `weather_data` 数据集，但需求文档中仅在"P2阶段"规划了"价格-温度相关性"
- 完全没有辐射、风速、降水与价差的分析

**建议新增**：
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 气象因子与价差相关性 | 辐射/风速/温度/降水 vs 价差的分时段相关性 | P1 |
| 气象组合条件识别 | 自动识别"高辐射+高风速+无降水=大正差"等规律 | P2 |
| 气象条件预警 | 结合天气预报提示价差方向 | P2 |

---

### 5. 异常价差深度归因 ⭐⭐

**业务需求**：
> 当价差很大时（如|价差|>50元），需要详细分析原因，积累经验

**系统现状**：没有异常归因分析功能

**建议新增**：
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 预测偏差归因 | 负荷/光伏/风电预测与实际的偏差分析 | P1 |
| 备用容量分析 | 正负备用变化对价差的影响 | P1 |
| 检修信息影响 | 机组检修/故障导致的供应变化 | P2 |
| 综合归因报告 | 按贡献度排序的主要因素和经验总结 | P1 |

---

## 四、已有但需调整的分析功能

### 1. "供给曲线"散点图 → 降级

**现有需求**：
> X轴=电量，Y轴=价格的散点图，展示"边际成本曲线"

**问题**：
- 这是电力市场微观分析工具，对日前交易决策价值有限
- 业务文档完全没有提及此类分析
- 实现复杂度高，收益低

**建议**：降低优先级至 P2 或删除

---

### 2. "储能行为专题" → 删除

**现有需求**：
> 验证储能是否在价高时放电、价低时充电

**问题**：
- 这是**储能运营商视角**的分析，不是售电公司核心需求
- 业务文档的储能策略是基于**价差套利机会**，而非储能行为分析

**建议**：
- 保留"储能套利机会分析（简化版）"
- 删除"储能行为专题"

---

### 3. "新能源渗透率 vs 价格" → 替换

**现有需求**：
> 散点图：X轴=新能源渗透率，Y轴=价格

**问题**：
- 过于学术化，对日前交易决策的直接指导有限
- 业务文档关注的是**新能源出力偏差**对价差的影响

**建议**：替换为"新能源出力偏差 vs 价差"的分析

## 五、需求优化建议汇总

### 5.1 优先级重排

| 原优先级 | 功能 | 调整建议 | 新优先级 |
|---------|------|---------|---------|
| 未规划 | **时段价差概率统计** | ⭐ 新增核心功能（最重要） | **P0** |
| 未规划 | **竞价空间分析** | ⭐ 新增核心功能 | **P0** |
| 未规划 | **边界条件敏感性分析** | ⭐ 新增核心功能 | **P0** |
| 未规划 | **异常价差深度归因** | 新增增强功能 | **P1** |
| P2 | 价格-温度相关性 | 扩展为气象因子分析 | P1 |
| P0 | 供给曲线散点图 | 降低优先级 | P2 |
| P1 | 储能行为专题 | 删除或降级 | P3 |
| P1 | 新能源渗透率分析 | 替换为出力偏差分析 | P1 |

---

### 5.2 新增数据需求

1. **竞价空间**：需要计算并存储
   - 公式：`竞价空间 = 系统负荷 - 新能源出力 - 非市场化机组出力`
   - 数据源：`actual_operation.system_load`、`real_time_generation`

2. **边界条件预测偏差**：
   - 负荷偏差 = 实际负荷 - 预测负荷
   - 光伏偏差 = 实际光伏 - 预测光伏
   - 风电偏差 = 实际风电 - 预测风电
   - 数据源：`daily_release` (预测) + `real_time_generation` (实际)

3. **日期类型标记**：
   - 工作日 / 周末 / 节假日
   - 用于分类统计价差概率

4. **气象数据关联**：
   - 将 `weather_data` 与价格数据进行时间关联
   - 需要的字段：`shortwave_radiation`、`wind_speed_10m`、`apparent_temperature`、`precipitation`

5. **检修信息**：
   - 新增 `maintenance_info` 集合
   - 用于异常价差归因分析

## 六、结论

### 6.1 优化方向

1. **聚焦日前交易决策** - 所有分析最终输出"做多/做空/观望"建议
2. **时段价差概率为核心** - 这是日前交易套利的直接依据
3. **分日类型统计** - 工作日/周末/节假日策略差异大
4. **多因素归因** - 理解价差驱动机制，提升预测准确性
5. **简化学术化功能** - 供给曲线、渗透率分析等可降级

### 6.2 预期效果

让操作员能快速回答以下核心问题：

**日前交易决策**：
- "这个时段历史上正差概率是多少？应该做多还是做空？"
- "工作日和周末的策略有什么差异？"
- "哪些时段是高确定性可以操作的？置信度如何？"

**风险评估**：
- "今天竞价空间多少？价格波动风险高不高？"
- "当前气象条件对价差有什么影响？"
- "这个异常价差是什么原因造成的？"

**策略优化**：
- "本月整体正负差规律如何？策略需要调整吗？"
- "哪些因子对价差影响最大？应该重点关注什么？"

### 6.3 核心指标

模块成功的关键指标：
- 高确定性时段识别准确率 > 80%
- 操作员日前交易决策效率提升 50%
- 套利成功率（实际价差方向与预判一致）> 70%
