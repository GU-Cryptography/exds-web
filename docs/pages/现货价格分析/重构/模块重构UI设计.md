# 现货价格分析模块 - UI设计文档

> 版本: v2.1
> 更新时间: 2025-11-23
> 设计理念: 日前交易决策导向、价差概率为核心、操作建议明确

---

## 零、业务背景

### 交易规则

- **中长期交易**：占80%，提前锁定价格和电量
- **现货交易**：占20%，日前交易报量不报价，按实时价格结算

### 套利逻辑

| 价差判断 | 操作策略 | 套利逻辑 | 颜色标识 |
|---------|---------|---------|---------|
| 日前 > 实时（正差） | **做空**（卖出电量） | 高价卖出 → 低价结算 | 🔴 红色 |
| 日前 < 实时（负差） | **做多**（买入电量） | 低价买入 → 高价结算 | 🟢 绿色 |

### 模块核心目标

**判断各时段正负价差的历史概率，输出明确的操作建议（做多/做空/观望），指导日前交易决策。**

## 八、移动端适配要点

### 8.1 通用原则

1. **面包屑标题**: 所有页面在移动端显示 `一级菜单 / 二级菜单`

## 十、设计总结

本UI设计遵循以下原则：

1. **决策导向**: 每个模块输出明确的策略建议
2. **信息分层**: 核心指标→图表→详情→建议
3. **一致性**: 统一的颜色编码、交互模式
4. **响应式**: 桌面端和移动端都能良好使用
5. **可操作性**: 策略建议具体可执行

预期效果：
- 交易员打开页面1分钟内获得今日策略建议
- 策略分析师能快速识别月度/周度规律
- 风险管理能实时监控竞价空间和边界条件
