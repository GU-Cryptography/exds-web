# 储能套利分析 (Energy Storage Arbitrage Analysis) - 业务需求文档

## 一、模块定位与目标

### 1.1 核心目标

通过分析现货市场的日内价格波动，为独立储能运营商提供：
- **套利机会识别**：自动识别每天的充放电最优时段组合
- **收益测算**：计算不同充放电策略的理论收益
- **策略评估**：对比"一充一放"vs"两充两放"的收益差异
- **历史回测**：评估策略在历史周期内的稳定性和可行性

### 1.2 典型应用场景

**场景1：日前策略制定**
- 问题：明天应该在哪些时段充电、哪些时段放电？
- 分析：基于日前价格预测，计算最优充放电组合
- 输出：充电时段列表、放电时段列表、预期收益

**场景2：实时策略调整**
- 问题：实时价格与日前偏离，是否需要调整充放电计划？
- 分析：对比实时价格的实际套利空间 vs 日前预期
- 输出：实际收益、偏差分析、策略调整建议

**场景3：历史收益回测**
- 问题：过去30天如果按"两充两放"策略运行，能赚多少钱？
- 分析：计算每天的最优策略和理论收益
- 输出：累计收益、日均收益、最佳/最差收益日

### 1.3 储能运营基本假设

| 参数 | 默认值 | 说明 | 可配置 |
|------|--------|------|--------|
| 储能容量 | 100 MWh | 电池总容量 | ✅ |
| 充电功率 | 25 MW | 最大充电功率 | ✅ |
| 放电功率 | 25 MW | 最大放电功率 | ✅ |
| 充电效率 | 95% | 充电过程损耗 | ✅ |
| 放电效率 | 95% | 放电过程损耗 | ✅ |
| 综合效率 | 90.25% | 充放电综合效率 (0.95×0.95) | 自动计算 |
| 最小SOC | 10% | 最低电量状态 | ✅ |
| 最大SOC | 90% | 最高电量状态 | ✅ |
| 可用容量 | 80 MWh | 可用电量 (90%-10%) | 自动计算 |

**约束条件**：
1. 单次充电/放电时长：至少4个时段 (1小时)
2. 充放电间隔：至少2个时段 (30分钟) 用于状态切换
3. 日内SOC守恒：每天结束时SOC回到初始状态 (如50%)

## 二、核心业务需求

### 需求1：每日最优充放电策略识别

**业务问题**：给定某一天的96点价格，如何找到最优的充放电时段组合？

#### 1.1 一充一放策略

**定义**：每天执行一次充电、一次放电

**算法逻辑**：
```python
# 伪代码
def find_one_charge_one_discharge(prices_96):
    """
    找到一充一放的最优组合
    """
    max_profit = 0
    best_strategy = None

    # 遍历所有可能的充电起始时段
    for charge_start in range(0, 93):  # 至少留4个点充电
        charge_end = charge_start + 4  # 充电1小时 (4个15分钟)

        # 充电电量 (考虑效率)
        charge_energy = 充电功率 × 1小时 × 充电效率  # MWh
        charge_cost = sum(prices_96[charge_start:charge_end]) / 4 × charge_energy

        # 遍历所有可能的放电起始时段 (必须在充电后)
        for discharge_start in range(charge_end + 2, 93):  # 间隔30分钟
            discharge_end = discharge_start + 4

            # 放电电量 (考虑效率)
            discharge_energy = charge_energy × 放电效率
            discharge_revenue = sum(prices_96[discharge_start:discharge_end]) / 4 × discharge_energy

            # 计算净收益
            profit = discharge_revenue - charge_cost

            if profit > max_profit:
                max_profit = profit
                best_strategy = {
                    'charge_start': charge_start,
                    'charge_end': charge_end,
                    'discharge_start': discharge_start,
                    'discharge_end': discharge_end,
                    'profit': profit,
                    'charge_avg_price': sum(prices_96[charge_start:charge_end]) / 4,
                    'discharge_avg_price': sum(prices_96[discharge_start:discharge_end]) / 4,
                    'price_spread': discharge_avg_price - charge_avg_price
                }

    return best_strategy
```

**输出指标**：
- 充电时段：起止时间、平均价格、充电量、充电成本
- 放电时段：起止时间、平均价格、放电量、放电收入
- 净收益 = 放电收入 - 充电成本
- 价差 = 放电均价 - 充电均价
- 单位收益 = 净收益 / 可用容量 (元/MWh)

#### 1.2 两充两放策略

**定义**：每天执行两次充电、两次放电

**典型模式**：
- **第一轮**：凌晨充电 (00:00-04:00) → 上午放电 (08:00-12:00)
- **第二轮**：中午充电 (12:00-14:00) → 晚上放电 (18:00-22:00)

**算法逻辑**：
```python
def find_two_charge_two_discharge(prices_96):
    """
    找到两充两放的最优组合
    复杂度：O(n^4)，需要优化
    """
    max_profit = 0
    best_strategy = None

    # 第一轮充放电
    for c1_start in range(0, 40):  # 第一次充电不能太晚
        c1_end = c1_start + 4

        for d1_start in range(c1_end + 2, 50):  # 第一次放电
            d1_end = d1_start + 4

            # 第二轮充放电
            for c2_start in range(d1_end + 2, 85):
                c2_end = c2_start + 4

                for d2_start in range(c2_end + 2, 93):
                    d2_end = d2_start + 4

                    # 计算总收益
                    profit = calculate_two_round_profit(
                        prices_96, c1_start, c1_end, d1_start, d1_end,
                        c2_start, c2_end, d2_start, d2_end
                    )

                    if profit > max_profit:
                        max_profit = profit
                        best_strategy = {
                            'round1': { 'charge': ..., 'discharge': ... },
                            'round2': { 'charge': ..., 'discharge': ... },
                            'total_profit': profit
                        }

    return best_strategy
```

**优化策略** (降低复杂度)：
1. **启发式搜索**：
   - 第一次充电限定在00:00-06:00 (凌晨低价)
   - 第一次放电限定在08:00-12:00 (上午高价)
   - 第二次充电限定在12:00-15:00 (午间低价)
   - 第二次放电限定在18:00-23:00 (晚高峰)

2. **动态规划**：
   - 将问题分解为两个独立的"一充一放"子问题
   - 约束：第二轮的起始SOC = 第一轮结束SOC

**输出指标**：
- 第一轮收益、第二轮收益、总收益
- 两轮收益贡献占比
- 对比一充一放的增量收益
- 增量收益率 = (两充两放收益 - 一充一放收益) / 一充一放收益

### 需求2：周期内套利收益分析

**业务问题**：过去30天，每天的最优策略是什么？累计收益多少？

**分析维度**：

#### 2.1 日维度收益统计

对于周期内的每一天，计算：

| 字段 | 说明 |
|------|------|
| 日期 | YYYY-MM-DD |
| 一充一放收益 | 元 |
| 两充两放收益 | 元 |
| 增量收益 | 两充两放 - 一充一放 |
| 增量收益率 | 增量收益 / 一充一放收益 × 100% |
| 最优策略 | "一充一放" 或 "两充两放" |
| 充电总成本 | 元 |
| 放电总收入 | 元 |
| 充电均价 | 元/MWh |
| 放电均价 | 元/MWh |
| 价差 | 元/MWh |
| 充电总量 | MWh |
| 放电总量 | MWh |

#### 2.2 聚合统计指标

| 指标 | 公式 | 业务含义 |
|------|------|---------|
| 累计收益 | Σ(daily_profit) | 周期总收益 |
| 日均收益 | 累计收益 / 天数 | 平均每天赚多少 |
| 最大单日收益 | max(daily_profit) | 最佳收益日 |
| 最小单日收益 | min(daily_profit) | 最差收益日 |
| 收益标准差 | std(daily_profit) | 收益稳定性 |
| 盈利天数 | count(profit > 0) | 有多少天赚钱 |
| 盈利概率 | 盈利天数 / 总天数 | 赚钱概率 |
| 两充两放优势天数 | count(两充两放 > 一充一放) | 两充两放更优的天数 |
| 两充两放优势概率 | 优势天数 / 总天数 | 多少情况下两充两放更好 |
| 平均增量收益 | mean(增量收益) | 两充两放平均多赚多少 |
| ROI | 累计收益 / (储能容量 × 平均价格) | 投资回报率估算 |

#### 2.3 时段偏好分析

**问题**：充电和放电更倾向于发生在哪些时段？

**分析方法**：
- 统计周期内每个时段被选为"充电时段"的次数
- 统计周期内每个时段被选为"放电时段"的次数
- 生成"充电热力图"和"放电热力图"

**业务价值**：
- 识别稳定的低价充电窗口
- 识别稳定的高价放电窗口
- 指导长期储能运营策略

**图表**：并排箱线图，对比两种策略的日收益分布

**分析**：
- 中位数：哪种策略的中位收益更高？
- 离散度：哪种策略更稳定？
- 离群值：是否存在极端收益/亏损日？

#### 4.3 策略切换建议

**逻辑**：
```python
if 两充两放收益 > 一充一放收益 × 1.1:  # 增量超过10%
    推荐策略 = "两充两放"
    理由 = "显著增加收益"
elif 两充两放收益 < 一充一放收益 × 0.95:  # 反而降低5%以上
    推荐策略 = "一充一放"
    理由 = "两充两放无优势，降低运营复杂度"
else:
    推荐策略 = "一充一放"
    理由 = "增量收益不显著，优先简单策略"
```

### 4.1 数据来源

| 数据 | 集合名称 | 字段 | 说明 |

### 4.2 计算复杂度

- **一充一放**：O(n²) ≈ 96² = 9,216 次计算/天
- **两充两放 (暴力搜索)**：O(n⁴) ≈ 96⁴ = 8,493万次计算/天 ❌ 不可行
- **两充两放 (启发式)**：O(n²) × 2 ≈ 1.8万次计算/天 ✅ 可行

**优化建议**：
1. 后端预计算并缓存结果
2. 使用动态规划或贪心算法减少搜索空间
3. 限制充放电时段候选范围 (如凌晨、上午、中午、晚上四个时段)

## 六、风险与限制

### 6.1 理论收益 vs 实际收益

**重要免责说明**：
- 本分析基于**历史价格**回测，假设可以完美预知价格
- 实际运营中，需要基于**预测价格**制定策略，存在预测误差
- 未考虑以下实际成本：
  - 电池损耗 (循环寿命)
  - 辅助服务成本 (调频、备用)
  - 电力交易手续费
  - 运维成本

**建议**：
- 在UI上明确标注"理论最优收益"
- 建议用户将实际收益打7-8折作为保守估计

### 6.2 价格波动风险

- 历史最优策略不代表未来最优
- 极端天气、政策变化可能导致价格模式改变
- 建议结合"现货趋势分析"模块评估价格稳定性

### 6.3 电网约束

- 未考虑并网功率限制
- 未考虑电网调度指令
- 实际运营需符合电网调度要求

## 九、附录：术语表

| 术语 | 定义 |
|------|------|
| 套利 (Arbitrage) | 利用不同时段价格差异赚取利润 |
| SOC (State of Charge) | 电池电量状态，百分比表示 (0-100%) |
| 充电效率 | 充电过程中实际存储的电量占充电电量的比例 |
| 放电效率 | 放电过程中实际释放的电量占存储电量的比例 |
| 综合效率 | 充放电循环的整体效率，= 充电效率 × 放电效率 |
| 一充一放 | 每天执行一次充电、一次放电的策略 |
| 两充两放 | 每天执行两次充电、两次放电的策略 |
| 理论最优收益 | 假设完美预知价格情况下的最大收益 |
| 价差 | 放电均价 - 充电均价 |
| 单位收益 | 净收益 / 充放电电量，单位：元/MWh |
| 增量收益 | 两充两放收益 - 一充一放收益 |
| 增量收益率 | 增量收益 / 一充一放收益 × 100% |
