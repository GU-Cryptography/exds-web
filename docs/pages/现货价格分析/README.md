## 📊 核心功能亮点

### 1. 现货趋势分析

**核心价值**：
- ✅ 识别价格、价差的长期趋势
- ✅ 挖掘周期性规律 (星期特性、时段特性)
- ✅ 分析价格与负荷、电源结构的相关性
- ✅ 评估市场波动性风险

**创新功能 - 储能套利机会分析 (简化版)**：
- 自动识别每天"上午"和"下午"两个时段的最大价差
- 上午时段：00:00-11:30 (凌晨低价充 → 上午高价放)
- 下午时段：11:30-23:30 (中午低价充 → 晚高峰放)
- 绘制30天价差趋势曲线，识别套利规律
- 算法简单 (O(n) 线性扫描)，实现快速
- 直接指导储能充放电决策

**技术特点**：
- 算法复杂度: O(n) 线性扫描 (96点/天)
- 30天计算量: ~2,880次 (毫秒级)
- 数据需求: 仅需96点价格数据

---

### 2. 时段特征挖掘

**核心价值**：
- ✅ 为每个时段 (48/96点) 提供统计特征
- ✅ 自动识别高确定性时段 (70%+一致性)
- ✅ 生成交易策略建议 (做多/做空/观望)
- ✅ 量化风险等级和置信度

**核心算法**：
```python
# 伪代码
for 每个时段:
    计算平均价格、价差、波动性
    计算正价差占比、负价差占比
    一致性评分 = max(正价差占比, 负价差占比)

    if 一致性评分 >= 70%:
        if 平均价差 > 10元:
            推荐策略 = "做空日前 (实时买入)"
        elif 平均价差 < -10元:
            推荐策略 = "做多日前 (日前买入)"
    else:
        推荐策略 = "观望"
```

**输出示例**：
> 时段15 (07:00-07:15):
> - 一致性评分: 85% (高)
> - 平均价差: -15元/MWh (实时价低于日前价)
> - 推荐策略: 做多日前
> - 置信度: 高
> - 风险等级: 低风险

## ⚠️ 关键技术挑战与解决方案

### 1. 算法性能优化

**挑战**: 时段特征挖掘需计算96点 × 30天 × 多指标

**解决方案**:
- 后端预计算并缓存结果
- 使用MongoDB聚合管道优化查询
- 日期范围变化时触发后台异步计算

### 2. 可视化复杂度

**挑战**: 热力图、箱线图的实现

**解决方案**:
- 热力图: 使用 `react-heatmap-grid` 或 Material-UI Table + 背景色
- 箱线图: 使用 `@nivo/boxplot` 或 Recharts `ComposedChart` 模拟

### 3. 数据完整性

**挑战**: 需要至少30天的完整96点数据

**解决方案**:
- 最小样本量检查: 前端提示"数据不足20天"
- 数据缺失处理: 后端返回有效天数，前端显示警告
- 建议周期: 30天为佳，90天最优

## 🎓 术语表

| 术语 | 定义 |
|------|------|
| VWAP | Volume-Weighted Average Price，加权平均价，反映真实购电成本 |
| TWAP | Time-Weighted Average Price，算术平均价，反映平均价格信号 |
| RT | Real-Time，实时市场 |
| DA | Day-Ahead，日前市场 |
| CV | Coefficient of Variation，变异系数，std/mean |
| 一致性评分 | 价差方向的稳定性，= max(正价差占比, 负价差占比)，范围[0.5, 1.0] |
| 做多日前 | 在日前市场买入，实时市场不操作 (预期实时价更高) |
| 做空日前 | 在日前市场不买或少买，等实时市场买入 (预期实时价更低) |
| 上午价差 | 00:00-11:30时段内，最高价 - 最低价 (约束: 先低后高) |
| 下午价差 | 11:30-23:30时段内，最高价 - 最低价 (约束: 先低后高) |
