# A3 角色场景回归报告（轻量版）

## 1. 使用说明

1. 适用场景：权限相关改动合入前、版本发布前、每周例行检查。
2. 角色范围：`viewer`、`analyst`、`business_admin`、`system_admin`。
3. 目标：快速发现“越权成功 / 误拦截 / 按钮漏绑 / 接口漏拦截”。
4. 结论口径：仅允许 `PASS`、`FAIL`、`N/A`。

## 3. 最小必测集（固定）

### 3.1 页面访问（R）

1. `viewer` 访问：客户档案、零售合同、零售套餐、用户与权限（应仅查看或拒绝进入管理能力）。
2. `analyst` 访问：分析预测类页面可进入，系统管理类按配置限制。
3. `business_admin` 访问：业务模块可进入，系统管理受限。
4. `system_admin` 访问：系统管理可进入。

### 3.2 按钮可用（B）

1. 客户档案：新增、编辑、删除。
2. 零售合同：新增、导入、删除。
3. 零售套餐：新增、导入/复制、删除。
4. 负荷数据诊断：重新聚合。
5. 结算：重新结算。
6. 用户与权限：新建用户、分配角色、重置密码、删除用户。

### 3.3 写请求结果（Q/S）

1. 无 `edit` 权限角色：写请求应前置拦截或 403，不可成功写入。
2. 有 `edit` 权限角色：对应写请求应成功。
3. 例外权限：删除/重算类接口必须同时满足例外权限。
