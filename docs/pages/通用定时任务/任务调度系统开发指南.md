# 任务调度系统开发指南 (v1.0)

本项目使用 `APScheduler` 作为任务调度引擎，结合 MongoDB `task_execution_logs` 集合实现任务执行过程的持久化记录与监控。

## 1. 核心架构

- **调度引擎**: `APScheduler` (AsyncIOScheduler)
- **存储介质**: `MemoryJobStore` (内存任务存储，重启后重置)
- **执行日志**: `task_execution_logs` 集合
- **配置文件**: `webapp/scheduler/core.py`

## 2. 核心组件说明

### 2.1 调度器初始化 (`webapp/scheduler/core.py`)
负责调度器的实例化、任务注册以及生命周期管理（FastAPI startup/shutdown）。

### 2.2 任务日志记录器 (`webapp/scheduler/logger.py`)
提供 `TaskLogger` 静态类，统一记录任务的开始、结束、摘要、详情以及错误信息。

## 3. 开发新任务流程

### 第一步：在 `jobs/` 目录下编写业务逻辑
创建新文件如 `webapp/scheduler/jobs/my_new_job.py`，建议结构如下：

```python
from webapp.scheduler.logger import TaskLogger
from webapp.tools.mongo import DATABASE
from datetime import datetime

async def my_new_task_job():
    # 1. 记录任务开始
    task_id = await TaskLogger.log_task_start(
        service_type="web",
        task_type="my_task",
        task_name="我的新任务",
        trigger_type="schedule"
    )
    
    try:
        # 2. 业务逻辑实现
        # ... your code ...
        
        # 3. 记录任务成功
        await TaskLogger.log_task_end(
            task_id=task_id,
            status="SUCCESS",
            summary="成功处理了 X 条记录",
            details={"processed": X, "errors": 0}
        )
    except Exception as e:
        # 4. 记录任务失败
        await TaskLogger.log_task_end(
            task_id=task_id,
            status="FAILED",
            summary="任务执行出错",
            error={"message": str(e)}
        )
```

### 第二步：在 `core.py` 中注册任务
修改 `webapp/scheduler/core.py`:

```python
from webapp.scheduler.jobs.my_new_job import my_new_task_job

def setup_scheduler(app):
    # ... 现有任务 ...
    
    # 注册新任务
    scheduler.add_job(
        my_new_task_job,
        'cron',     # 或者 'interval'
        hour=2,     # 每天凌晨2点
        minute=0,
        id='my_new_task_id',
        replace_existing=True
    )
```

## 4. 最佳实践建议

### 4.1 增量执行逻辑
推荐使用"内存缓存 + 日期判断"的模式实现每天仅执行一次的任务，避免因重复触发或系统重启导致的重复计算。参考 `aggregation_jobs.py` 中的 `_is_executed_today` 实现。

### 4.2 错误处理与告警
- 任务内部必须使用 `try...except` 捕获异常。
- 关键失败可通过 `DATABASE['system_alerts']` 写入 P1 告警。

### 4.3 时间记录
- **一律使用系统本地时间** (`datetime.now()`)，避免时区混淆。

## 5. 日志查询
可通过查询 `task_execution_logs` 集合监控任务状态：
```javascript
db.task_execution_logs.find({ task_type: "load_aggregation" }).sort({ start_time: -1 })
```
