# 客户档案管理重构方案 & UI 设计书 (v3 - 最终版)

## 目标描述
根据 `customer_archives` 数据集结构 (v2) 重构客户管理模块，采用独立的详情页和编辑页设计，优化标签管理和筛选体验，并增加安全确认机制。

---

## UI 设计方案

### 1. 列表页 (List View)

#### 列定义
| 列名 | 说明 |
|------|------|
| **客户名称** | 全称 + 简称 (可点击，打开详情弹窗) |
| **气象区域** | 关联的气象地名称 |
| **标签** | Chips 展示 (前3个 + [+N]) |
| **资产统计** | 户号数 / 电表数 / **计量点数** |
| **操作** | 查看、编辑、删除 (~~复制~~已移除) |

#### 交互
- **点击客户名称**: 打开 **详情弹窗 (Detail Dialog)**。
- **删除操作**: 弹出密码确认框，需输入当前登录密码后方可执行删除。

#### 标签筛选器 (Tag Filter)
- **外观**: 组合框，已选标签以 Chip 展示。
- **下拉面板**:
    - 搜索框 + 分组展示 (风险/用电/...)。
    - 点击标签即选中/取消。
    - **仅支持选择已有标签**，不在此处新增。
- **逻辑**: 多选 (OR)。
- **重置按钮**: 筛选区域末尾提供 "重置" 按钮，一键清空筛选条件。

### 2. 详情弹窗 (Detail Dialog) - **独立设计**

**目的**: 只读展示客户完整信息，并关联显示该客户的零售合同历史。

#### 内容区域
- **基础信息**: 客户全称、简称、气象区域、经纬度、客户经理。
- **标签展示**: 只读 Chip 列表。
- **户号与资产**: 只读列表展示 (户号 -> 电表/计量点)。
- **关联合同记录 (新增)**: 表格展示该客户的所有 `retail_contracts`。
    - 列: 合同名称, 套餐名称, 购电区间, **签约电量**。
    - 点击可跳转至合同详情页 (或弹窗展示快照)。

#### 操作按钮
- `[ 编辑 ]`: 打开 **编辑弹窗**。
- `[ 关闭 ]`。

### 3. 编辑弹窗 (Edit Dialog) - **独立设计**

**目的**: 编辑客户信息。**与详情弹窗分离，不复用**。

#### A. 基础信息区
```text
+---------------------------+  +---------------------------+
| 客户全称*: [ 江西XX公司  ] |  | 客户简称*: [ 江西制造    ] |
+---------------------------+  +---------------------------+

+------------------------------+  +------------+  +------------+
| 气象区域*: [ 南昌 (Nanchang) v] |  | 经度 (只读) |  | 纬度 (只读) |
+------------------------------+  +------------+  +------------+
(点击下拉无匹配时显示: [📍+ 新增气象区域...])
```
- **气象区域选择**:
    - 只读联动经纬度，**不允许手动修改坐标**。
    - 若无匹配区域，下拉底部显示 **"➕ 新增气象区域..."** 按钮，点击后弹出 **新建 Weather Location** 表单 (同 Weather Data 页表单)，提交后自动选中新记录。

#### B. 标签管理区 (Tag Manager)
```text
[ VIP (x) ] [ 生产中 (x) ]  [ + 添加标签 ]
```
- 点击 `[ + 添加标签 ]` 弹出 Popover (分组选择/搜索)。
- **新增标签**: 若搜索词在 `customer_tags` 集合中不存在，显示 "**➕ 新建标签: [xxx]**"，点击后临时创建并添加。

#### C. 户号与资产区 (Accounts & Assets)
- 可编辑的户号列表。
- 每个户号下可增删电表和计量点。

#### 操作按钮
- `[ 取消 ]` `[ 保存 ]`。

## 移动端设计 (Mobile Design)

**设计原则**: 参考原页面设计，移动端采用**路由导航**而非弹窗，使用**卡片式列表**展示数据。

### 1. 移动端列表页
- **布局**: 卡片式列表，每个客户一张卡片。
- **卡片内容**:
    - **标题行**: 客户名称 (可点击，使用路由跳转到详情页)。
    - **信息区**: 气象区域、标签 (Chips)。
    - **统计行**: 户号数 / 电表数 / 计量点数。
    - **操作行**: 查看、编辑、删除 (IconButton)。
- **筛选**: 可折叠的筛选卡片 (点击图标展开/收起)。
- **新增按钮**: 位于筛选卡片下方，列表上方。

### 2. 移动端详情页 (路由: `/customer/profiles/view/:id`)
- **顶部**: 返回按钮 + "客户详情" 标题。
- **内容**: 垂直滚动展示，与桌面端详情弹窗内容一致。
- **操作**: 底部固定 "编辑" 按钮。

### 3. 移动端编辑页 (路由: `/customer/profiles/edit/:id`)
- **顶部**: 返回按钮 + "编辑客户" 标题。
- **内容**: 表单垂直布局，与桌面端编辑弹窗内容一致。
- **操作**: 底部固定 "保存" 按钮。

### 4. 移动端新增页 (路由: `/customer/profiles/create`)
- **顶部**: 返回按钮 + "新增客户" 标题。
- **内容**: 空表单，与编辑页布局相同。

## 建议变更计划

### 后端
- **customer.py**: 移除 `status` 及旧字段，新增 `tags`, `manager`, `source`，更新 `location` 逻辑。
- **v1.py**: 新增 `/customers/{id}/delete` 接口，需传入密码验证;或在现有 DELETE 接口中增加 `password` 请求体校验。

### 前端
1.  **TagFilter 组件**: 多选筛选器，仅支持选择已有标签。
2.  **TagSelector 组件 (Popover)**: 编辑时的标签选择器，支持新增标签。
3.  **CustomerDetailDialog**: 新建/重构，展示只读详情 + 关联合同。
4.  **CustomerEditorDialog**: 重构，不复用 Detail，支持气象区域内联新增。
5.  **DeleteConfirmDialog**: 带密码输入的删除确认。
6.  **WeatherLocationCreateDialog**: 新增气象位置的快速表单 (或复用 Weather Data 页组件)。

## 验证计划
1.  列表页标签筛选器能正确筛选包含指定标签的客户。
2.  点击客户名称打开详情弹窗，显示关联合同。
3.  详情弹窗点击"编辑"打开独立编辑弹窗。
4.  编辑弹窗选择气象区域后，经纬度自动填充且为只读。
5.  编辑弹窗点击"新增气象区域"可创建新 weather_location 并自动选中。
6.  编辑弹窗标签管理中可临时新增标签。
7.  删除客户需输入密码确认。
