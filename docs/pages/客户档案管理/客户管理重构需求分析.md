# 客户管理重构需求分析

## 1. 客户档案模块定位
- **功能**: 仅用于新增或编辑客户的基本档案信息。
- **关联**: 不关联零售合同信息，不维护客户状态。

## 2. 零售合同管理模块定位
- **功能**: 不支持直接新增或修改，只能通过导入交易中心平台订单信息来创建。
- **关联处理**: 
    - 导入后，系统将尝试关联客户和套餐信息。
    - 如果未关联到现有客户或套餐，系统将根据零售合同内的客户信息和套餐信息名称，在客户档案和零售套餐页面自动新建对应记录。
    - 系统将提示用户对新建的客户档案和零售套餐记录进行信息完善。

## 3. 菜单结构调整
- 将原有的“客户档案”和“零售合同”模块移动至“基础数据”一级菜单下。
- 将原有的一级菜单“客户管理”更名为“客户分析”。

## 4. 客户分析模块功能
- **定位**: 基于基础数据（客户档案、合同、套餐信息）、客户负荷数据以及未来的结算结果，进行客户的全面分析。
- **子模块**:
    - 客户统计面板
    - 客户360详情
    - 客户负荷分析
    - 临时客户诊断
