# 零售侧结算实施方案

## 一、业务背景

零售侧结算目标：根据每个客户的零售合同中绑定的套餐定价模型，结合每日峰谷时段和用户 48 时段电量，计算用户侧的零售电费。

结算模式分两种场景：
- **日常预结算**：每日计算，使用当日实时数据（用于展示当日预结算均价和电费）。
- **月度正式结算**：次月生成，使用全月汇总数据（用于生成最终结算单）。

---

## 二、2026年合同与套餐分析

### 2.1 套餐分类汇总

2026年共 41 个有效合同，涉及 16 种套餐，主要分为以下三大类：

#### 类别A：平均上网电价分成类（34个合同）
- **定价逻辑**：基于月度平均上网电价（参考价）进行价差扣减或分成。
- **参考价**：`price_sgcc.avg_on_grid_price`（国网代购电价中的平均上网电价）。
- **模型代码**：`price_spread_simple_price_time`。

#### 类别B：上限分成类（5个合同）
- **定价逻辑**：基于官方最高限价进行分成。
- **基准价**：414.3 × 1.2 = **0.49716 元/kWh**。
- **模型代码**：`price_spread_simple_price_time`。

#### 类别C：固定价+联动类（2个合同）
- **定价逻辑**：固定分时价格与现货均价联动。
- **联动标的**：当日实时现货均价（48点聚合）。
- **模型代码**：`fixed_linked_price_time`。

## 三、核心业务规则

### 3.1 定价公式

#### A. 价差分成类
1.  **取参考价基准值**：
    - `market_monthly_avg` → 取结算月 `avg_on_grid_price`。
    - `upper_limit_price` → 固定为 0.49716。
2.  **分时展开**：按 **1.8:1.6:1:0.4:0.3** 比例将基准值展开为 5 个时段参考价。
3.  **扣减计算**：`价格[时段] = 参考价[时段] - 约定价差 × 分成比例 + 浮动价`。
4.  **比例调节**：执行 463 号文件规定的比例保护逻辑。

#### B. 固定价+联动类
1.  **固定部分**：读取套餐配置的 5 时段固定价格。
2.  **联动部分**：获取当日实时现货 96 点数据，聚合为 48 点，按时段类型计算均价。
3.  **混合计算**：`最终价 = 固定价 × (1 - 联动比例) + 联动价 × 联动比例`。

### 3.2 价格比例调节规则 (463号文)

标准比例：**尖峰1.8 : 高峰1.6 : 平段1 : 低谷0.4 : 深谷0.3**。

- **锚点**：以平段价为基准。
- **下限调节**：尖峰/高峰价格若低于标准比例，则强制上调至标准值。
- **上限调节**：低谷/深谷价格若高于标准比例，则强制下调至标准值。
- **单向逻辑**：若价格已经比标准比例更优（尖峰更高或谷时更低），则保持不变。

### 3.3 计算流程

1.  **数据就绪检查**：确认当日 48 段电量、峰谷时段定义、现货价格均已到位。
2.  **合同规则解析**：识别用户生效合同、套餐参数及参考价基准。
3.  **时段定价**：根据定价模型计算 5 个时段的结算单价。
4.  **费用核算**：将 48 段电量与对应时段单价相乘，汇总得出日总电费。
5.  **聚合存储**：计算日加权均价，并存储明细到 `retail_settlement_daily` 集合。

## 四、数据结构 (retail_settlement_daily)

集合存储详细的日结算结果，支持按日、按客户查询。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `customer_id` | String | 客户唯一标识 |
| `date` | String | 结算日期 (YYYY-MM-DD) |
| `package_name` | String | 套餐名称 |
| `final_prices` | Object | 最终执行的 5 时段结算价 (尖/峰/平/谷/深) |
| `total_load_mwh` | Number | 当日总电量 (MWh) |
| `total_fee` | Number | 当日总电费 (元) |
| `avg_price` | Number | 当日加权均价 (元/kWh) |
| `tou_summary` | Object | 分时段电量电费汇总 |
| `period_details` | Array | 48 点结算明细 (包含每时段电量、单价、电费) |

**索引信息**:
- 唯一索引: `customer_id`, `date`, `settlement_type`

---

## 五、API 定义

### 5.1 触发重算
- **POST** `/api/v1/retail-settlement/calculate`
- 参数：`date` (日期), `force` (是否强制重算)

### 5.2 查询日结算
- **GET** `/api/v1/retail-settlement/daily`
- 参数：`start_date`, `end_date`, `customer_id`
