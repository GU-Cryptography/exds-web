# 意向客户诊断模块技术方案

## 1. 文档目标

本文档用于指导“意向客户诊断”模块的前后端开发与联调，覆盖以下内容：

- 页面信息架构与 UI 初步方案（含 ASCII 线框）
- 核心业务流程与计算口径
- 数据集结构设计（MongoDB）
- API 设计与前后端交互契约
- MVP 实施范围与验收清单

模块挂载位置：`客户管理 > 意向客户诊断`

---

## 2. 模块边界与业务目标

### 2.1 业务目标

针对非签约客户（意向客户），基于其导入的历史电表示数与负荷数据：

1. 生成可用于结算分析的 48 点负荷曲线
2. 自动识别负荷覆盖的已结算月份
3. 计算批发侧月度成本与单位成本
4. 对比不同零售套餐下的月度毛利与价差
5. 给出推荐套餐（主推+备选）

### 2.2 模块范围

本期范围（MVP）：

- 客户选择/新增
- 负荷数据导入、断点检测、48 点曲线计算
- 批发侧月结成本计算
- 多套餐并排测算与推荐

后续范围（V2）：

- 负荷特性分析（峰谷比、负荷率、波动性）
- 分析报告导出（PDF/Excel）

## 3. 页面信息架构

### 3.1 顶部公共区

- 客户选择下拉框（仅意向客户）
- 新增客户按钮
- 删除客户按钮（仅对可删除客户显示，点击后需二次输入密码确认）
- 客户信息摘要仅展示：数据导入时间

### 3.2 三个 Tab

1. `客户负荷数据`
- 电表数据一体化导入与自动聚合
- 负荷汇总信息与完整性信息
- 按月显示日负荷、按日显示 48 时段负荷

2. `月度模拟结算`
- 自动识别所有“负荷已覆盖且批发月结已完成”的月份
- 批发侧一键批量计算全部满足月份
- 上半部分展示批发侧月度汇总表与单月明细
- 下半部分预留零售侧多套餐多版本测算区

3. `零售套餐对比`
- 手工选择多个套餐
- 并排比较毛利/价差
- 自动给出推荐套餐

## 7. 核心计算口径

### 7.1 48 点曲线计算

输入：按时间排序的累计电表示数。

规则：

1. 相邻时点差分得到区间电量
2. 读数倒退判定为异常，写入质量问题表
3. 缺失时段记为断点，不自动补值（MVP）
4. 聚合到每日 48 点（`point=1..48`）

### 7.2 批发侧月度成本

按月汇总：

- `total_cost = energy_cost + deviation_cost + other_cost`
- `unit_cost = total_cost / total_load_kwh`

说明：批发价格与规则来源于已月结数据及现有结算规则服务。

### 7.3 套餐收益与价差

- `gross_profit = retail_revenue - wholesale_cost`
- `spread = retail_unit_price - wholesale_unit_cost`
- `gross_margin = gross_profit / retail_revenue`

推荐评分建议：

- `recommend_score = 收益得分*0.6 + 稳定性得分*0.3 - 风险惩罚*0.1`

## 8. API 设计（草案）

### 8.1 客户相关

1. `GET /api/v1/intent-customers`
2. `POST /api/v1/intent-customers`
3. `GET /api/v1/intent-customers/{customer_id}`

### 8.2 负荷导入与校验

1. `POST /api/v1/intent-customers/import-preview`
- 入参：客户名 + 文件列表
- 出参：户号、电表、日期范围识别结果及倍率待补录项

2. `POST /api/v1/intent-customers/import-and-aggregate`
- 入参：客户名/客户ID + 文件列表 + 倍率配置
- 出参：导入摘要、聚合摘要、覆盖日期范围

3. `GET /api/v1/intent-customers/{customer_id}/load-data/summary`
- 出参：数据范围、覆盖天数、缺失天数、完整率、日均电量等

### 8.3 批发月结测算

1. `GET /api/v1/intent-customers/{customer_id}/wholesale/months-covered`
2. `POST /api/v1/intent-customers/{customer_id}/wholesale/recalculate`
3. `GET /api/v1/intent-customers/{customer_id}/wholesale/monthly-results`

### 8.4 套餐对比测算

1. `POST /api/v1/intent-customers/{customer_id}/package-simulations`
- 入参：`months[]`, `package_ids[]`
- 出参：`simulation_id`, 推荐结果

2. `GET /api/v1/intent-customers/{customer_id}/package-simulations/{simulation_id}`

## 10. 关键交互与异常处理

1. 未选择客户时禁用 Tab 操作区
2. 导入失败时显示错误明细（行号+原因）
3. 计算中禁用“重新计算/开始测算”按钮，防重复提交
4. 图表刷新采用覆盖层，保持全屏状态不丢失
5. 套餐不足 2 个时允许测算，但不触发“并排推荐”

## 11. 实施计划（MVP）

### 11.1 本轮目标（优先实现）

本轮只实现以下范围，不进入 Tab2 / Tab3：

- 顶部公共区调整
- `客户负荷数据` Tab 完整首版
- 删除客户密码确认交互

### 11.2 子任务拆分

#### 子任务 A：顶部公共区调整

- 调整客户摘要字段，只保留“数据导入时间”
- 在顶部区域增加“删除客户”按钮
- 明确删除按钮显示规则：
  - 未选择客户时隐藏或禁用
  - 无导入记录时，客户摘要显示“暂无导入记录”
  - 删除动作需二次确认，不允许误触直接删除
- 确认删除交互文案：
  - 首次提示：确认是否删除当前客户
  - 二次确认：输入当前登录密码后才允许提交
- 优先复用现有客户删除能力，避免新开一套删除协议

#### 子任务 B：顶部公共区数据接口梳理

- 明确客户下拉数据来源
- 明确“数据导入时间”的取值口径：
  - 取该客户最近一次成功导入的时间
  - 没有导入记录时返回空值，由前端展示“暂无导入记录”
- 明确删除成功后的页面行为：
  - 从客户下拉中移除已删除客户
  - 清空当前选中客户
  - 默认停留在 `客户负荷数据` Tab

#### 子任务 C：`客户负荷数据` Tab 信息架构细化

- 页面首版直接面向聚合后的负荷结果
- 页面核心内容收敛为 3 类：
  - 一体化导入入口
  - 负荷汇总与完整性信息面板
  - 月度日负荷图 + 日内 48 时段图

#### 子任务 D：数据导入与基础校验

- 支持一次性拖入该客户全部电表数据文件
- 自动识别户号、电表号、日期范围
- 用户补录倍率后，一次性完成导入与聚合
- 返回导入摘要与聚合摘要，不单独设计批次管理页面

#### 子任务 E：完整性分析与异常识别

- 参考 `CustomerLoadAnalysisPage.tsx` 中“用电特征标签与模式识别”面板布局
- 在该信息面板中展示：
  - 数据范围
  - 覆盖天数
  - 缺失天数
  - 完整率
  - 日均电量
  - 最大日/最小日
- 异常信息以摘要为主，不在首版单独展开原始断点明细表

#### 子任务 F：聚合结果展示

- 支持按月查看每日负荷曲线
- 支持按日查看 48 时段负荷曲线
- 两类图表均需支持全屏
- 支持月份切换与日期切换

#### 子任务 G：48 点曲线与完整性分析的边界控制

- `客户负荷数据` Tab 主体直接展示聚合结果，不再先展示原始电表表格
- 图表优先级：
  - 先实现按月日负荷图
  - 再实现按日 48 时段图
- 图表一旦实现，必须按现有规范接入 `useChartFullscreen`

#### 子任务 H：联调与验收

- 验证未选客户、无数据客户、已导入客户三种状态
- 验证删除客户需密码确认，且删除后页面状态正确回收
- 验证导入后顶部“数据导入时间”能同步刷新
- 验证 `客户负荷数据` Tab 在移动端可正常浏览表格和图表

1. 先完成顶部公共区静态结构与交互流程
