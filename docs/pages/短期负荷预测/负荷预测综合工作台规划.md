# 负荷预测模块重构方案 (精度与对比导向)

针对您的深度反馈，本方案将布局优化为“上下分层+指标内嵌”结构，强化总体与分项的联动，并明确负荷预测特有的“宽表”精度数据访问逻辑。

## 1. 核心设计原则

*   **单屏驾驶舱 (Single-Screen Cockpit)**：
    *   **高度锁定**: 桌面端强制使用 `100vh`（减去顶部导航）布局，确保所有核心指标、总体图表与客户详情在同一视野内，无需滚动。
    *   **弹性分层**: L2 (总体) 与 L3 (分项) 采用可调或固定比例分配高度（如 45%:55%），确保图表均具备足够的观测深度。
*   **上下文指标 (Contextual Metrics)**：根据选择日期的时态，展示“实测数据”或“历史可靠性”。

---

## 2. 页面规划：负荷预测综合工作台 (`LoadForecastWorkbench`)

### 2.1. L1: 控制区与核心指标 (Selection & Stats)
*   **双日期选择器**: 目标日期 + 发布版本下拉框。
*   **内置指标 (Indicators)**: 在控制面板右侧或图表顶部角落展示：
    *   **预测准确率 (WMAPE)**: 未来看史均，历史看实测。
    *   **日电量 (Daily Energy)**: 同时展示 **预测值** 与 **实际值** (历史日期)，方便量化对比。

### 2.2. L2: 总体负荷预测区 (Fixed Height ~40vh)
*   **紧凑图表设计**: 缩减非必要边距，将关键指标（WMAPE, 日电量）以 Overlay 浮窗形式置入图表右上角，释放纵向空间。
*   **视觉元素**: 保持 TOU 背景与 90% 置信区间。

### 2.3. L3: 客户联动工作区 (Fixed Height ~50vh)
*   **并列布局**: 左侧客户列表与右侧详情 Tab 横向铺开。
*   **左侧：客户导航 (Virtualized List)**
    *   **数据范围 (Scope)**: 默认加载 **本年度所有签约客户**。
    *   **展示策略**: 
        *   **一律使用客户简称 (Short Name)**: 为节省横向像素并提升识别效率，列表中仅展示客户简称。
        *   针对约 40 个客户，采用 **紧凑型列表 (Compact List)** + **局部滚动**。
        *   **一键搜索**: 顶部吸顶搜索框。
        *   **异常提示**: 对于合约已生效但预测数据缺失的客户，在列表中以灰色置灰并显示“缺失”提示，引导补测。
    *   **指标显示**: 列表显示客户名及其内嵌的 WMAPE 标签。
*   **右侧：详情 Tab**: 对应所选客户的曲线与修正面板。

*   **右下：详情工作组 (Details Group)**
    *   **Tab 1：客户曲线 (Chart Tab)**
        *   **全联动逻辑**: 曲线内容严格遵循全局“预测版本”下拉框。
        *   **细节组件**: 集成 **TOU 时段背景**。
        *   **注意**: 客户级数据 **不显示** 置信区间，仅展示预测线及实际线对比。
    *   **Tab 2：手工修正 (Adjustment Tab - 预留)**
        *   提供曲线拉拽、表格调整功能。
        *   **相似日参考**：在该 Tab 激活时，提供“最近类似天气”、“相同工作属性日”的智能参考日期选择。

## 3. 技术机制规划

### 3.1. 数据访问逻辑 (Wide-Table Structure)
*   **精度获取**：负荷预测不同于价格预测，不需要跨集合查询指标。直接读取 `load_forecast_results` 文档中的 `accuracy` 对象即可。
*   **字段映射**：
    *   `wmape` -> `accuracy.wmape_accuracy`
    *   `mae` -> `accuracy.mae`

### 3.2. 联动聚合逻辑
*   **触发场景**：用户在“手工修正”Tab 保存数据后，系统即刻执行重聚合任务，并刷新顶层的总体负荷曲线（AGGREGATE）。

### 3.2. 视觉表现优化
*   **Actual (实际)**：实黑线。
*   **Gap 0 (预测)**：深蓝实线。
*   **误差区间**：预测与实际间的填充色。

---

## 4. 验证计划

1.  **日期切换测试**：验证切换至昨日是否自动出现“实际 vs 预测”对比。
2.  **精度关联测试**：验证客户列表中的准确率数字是否与后端 `forecast_accuracy_daily` 集合一致。
3.  **聚合一致性**：验证手工修正客户 A 后，总体负荷曲线是否能立即反映出该变化。
