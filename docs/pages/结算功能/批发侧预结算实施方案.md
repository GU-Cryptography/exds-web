# 批发侧预结算模块实施方案

## 目标
实现**每日预结算**功能：每天计算 D-2（或 D-1）的批发侧结算费用，重点包括**电能量结算**（与平台数据比对）和**偏差回收考核**（本地模拟计算）。

---

## 1. 预结算数据集结构

### 1.1 集合名: `settlement_daily`

每日一条文档。**核心设计原则**：所有日汇总数据均为 **48个时段明细数据的统计计算结果** (Sum 或 Weighted Avg)。

#### A. 基础信息

| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `operating_date` | String | **[主键]** 结算日期 (YYYY-MM-DD) |
| `version` | Int | 计算版本号 |
| `created_at` | ISODate | 创建时间 |
| `updated_at` | ISODate | 更新时间 |

#### B. 分时明细 `period_details` (48点数组)

> **这是结算的核心**。数组包含 48 个对象，对应当日 48 个时段。

**单时段对象结构**:

| 分类 | 字段名 | 类型 | 描述 | 计算逻辑 |
| :--- | :--- | :--- | :--- | :--- |
| **时段** | `period` | Int | 时段号 (1-48) | - |
| **基础** | `mechanism_volume` | Float | 机制电量 (MWh) | `Q_Mechanism` |
| **中长期** | `contract.volume` | Float | 合同电量 | `Q_Contract` |
| | `contract.price` | Float | 合同价格 | `P_Contract` |
| | `contract.fee` | Float | 中长期差价费 | `Q_Contract * (P_Contract - P_RT)` |
| **日前** | `day_ahead.volume` | Float | 日前电量 | `Q_DA` |
| | `day_ahead.price` | Float | 日前价格 | `P_DA` |
| | `day_ahead.fee` | Float | 日前差价费 | `Q_DA * (P_DA - P_RT)` |
| **实时** | `real_time.volume` | Float | 实际用电量 | `Q_RT` |
| | `real_time.price` | Float | 实时价格 | `P_RT` |
| | `real_time.fee` | Float | 实时基础费 | `Q_RT * P_RT` |
| **汇总** | `total_energy_fee` | Float | 电能量电费 | `Fee_RT + Fee_DA + Fee_Contract` |
| | `energy_avg_price` | Float | 结算均价 | `Fee_Total / Q_RT` |
| **考核** | `contract_ratio` | Float | 签约比例 (%) | `(Q_Contract + Q_Mechanism) / Q_RT` |
| | `standard_value_cost` | Float | 标准值费用 | 根据 80/120 规则计算的模拟费用 |

#### C. 日汇总数据 (由 B 聚合)

> 对应平台文件：《现货结算--用电侧日结算信息.xls》
> **所有字段均由 `period_details` 累加或加权平均得出**

| 字段名 | 类型 | 描述 | 聚合逻辑 |
| :--- | :--- | :--- | :--- |
| `contract_volume` | Float | 中长期合同电量 | `Sum(period_details.contract.volume)` |
| `contract_avg_price` | Float | 中长期合同均价 | `Sum(Vol * Price) / Sum(Vol)` |
| `contract_fee` | Float | 中长期差价电费 | `Sum(period_details.contract.fee)` |
| `day_ahead_volume` | Float | 日前出清电量 | `Sum(period_details.day_ahead.volume)` |
| `day_ahead_fee` | Float | 日前差价电费 | `Sum(period_details.day_ahead.fee)` |
| `real_time_volume` | Float | 实际用电量 | `Sum(period_details.real_time.volume)` |
| `real_time_fee` | Float | 实时全电量电费 | `Sum(period_details.real_time.fee)` |
| `total_energy_fee` | Float | 电能量电费合计 | `Sum(period_details.total_energy_fee)` |

## 2. 核心业务逻辑 (已确认)

### 2.1 偏差回收考核 (标准值机制) - **分时考核**

> **关键规则更新**: 偏差考核需对每日 **48个时段** 分别独立计算，日累计费用为各时段费用之和。

- **考核指标 (分时)**: `K(t) = (Q_Contract(t) + Q_Mechanism(t)) / Q_RT(t)`
    - *注：分母为实时电量，分子由（中长期合同 + 机制电量）构成。*

- **计算逻辑 (单时段 t)**:
    1.  **若 0.8 <= K(t) <= 1.2**:
        - 标准值费用 `Cost_Std(t) = Cost_Actual(t)`
    2.  **若 K(t) < 0.8 (少签)**:
        - **按 80% 补足**: 缺口 `Q_Gap = 0.8 * Q_RT(t) - (Q_Contract(t) + Q_Mechanism(t))`
        - **补足价格**: `P_Market_Avg` (全市场中长期均价)
        - **标准值费用**: `Cost_Std(t) = Cost_Actual(t) + Q_Gap * (P_Market_Avg(t) - P_RT(t))`
    3.  **若 K(t) > 1.2 (多签)**:
        - **按 120% 剔除**: 目标允许签约量 `Q_Allowed = Max(0, 1.2 * Q_RT(t) - Q_Mechanism(t))`
        - **标准值费用**: `Cost_Std(t) = (Q_Allowed * (P_Company_Contract(t) - P_RT(t))) + Cost_DA(t) + Cost_RT(t)`

- **日累计**: `Daily_Recovery = Max(0, Sum(Cost_Std(t)) - Sum(Cost_Actual(t)))`
    - *注：采用日累计净额结算 (Daily Netting)，特定时段的正偏差可抵消负偏差。*

## 3. 数据来源映射 (已确认)

| 预结算字段 | 数据来源集合 / 服务 | 筛选/计算逻辑 |
| :--- | :--- | :--- |
| **实际用电量** ($Q_{RT}$) | `unified_load_curve` (LoadQueryService) | `strategy=MP_ONLY` (仅使用MP数据) |
| **机制电量** ($Q_{Mechanism}$) | `mechanism_energy_monthly` | 筛选: `month_str`, 计算: $Value_{Daily} = Value_{Monthly} / DaysInMonth$ |
| **实时现货价格** ($P_{RT}$) | `real_time_spot_price` | 字段: `avg_clearing_price`, **96点转48点**: 算术平均 `(P1+P2)/2` |
| **日前现货价格** ($P_{DA}$) | ① `day_ahead_spot_price`<br>② `day_ahead_econ_price` | ① 2026年1月及以前<br>② 2026年2月及以后。<br>**96点转48点**: 算术平均 |
| **日前申报电量** ($Q_{DA}$) | `day_ahead_energy_declare` | 字段: `energy_mwh` |
| **中长期合同** ($Q_{Contract}$) | `contracts_aggregated_daily` | 筛选: `entity="售电公司"`, `contract_type="整体"`, `contract_period="整体"`, `periods`数组 |
| **全市场合同均价** ($P_{Market\_Avg}$) | `contracts_aggregated_daily` | 筛选: `entity="全市场"`, `contract_type="整体"`, `contract_period="整体"`, `periods`数组 |

## 4. API 接口定义

### 4.1 触发日结算计算
- **URL**: `/api/v1/settlement/calculate`
- **Method**: `POST`
- **Summary**: 触发指定日期的批发侧预结算计算 (支持重算)
- **Request Body**:
    ```json
    {
      "date": "2026-02-01",  // 必填, 结算日期
      "force": false         // 选填, 是否强制重算 (默认 false)
    }
    ```
- **Response**:
    ```json
    {
      "code": 200,
      "message": "Calculation completed",
      "data": { ...SettlementDaily Object... }
    }
    ```

### 4.2 获取日结算数据
- **URL**: `/api/v1/settlement/daily`
- **Method**: `GET`
- **Summary**: 获取指定日期范围的日结算数据
- **Query Params**:
    - `start_date`: YYYY-MM-DD (必填)
    - `end_date`: YYYY-MM-DD (必填)
    - `include_details`: bool (可选, 默认 false) - **是否包含分时明细**。为 false 时仅返回日汇总数据 (Table C)，避免大数据传输。
- **Response**:
    ```json
    {
      "code": 200,
      "data": [
        {
          "operating_date": "2026-02-01",
          "total_energy_fee": 1234.5,
          ... (不包含 period_details 若 include_details=false) ...
        }
      ]
    }
    ```

## 5. 自动化调度 (定时任务)

### 5.1 任务逻辑
- **目标**: 每天自动检查并计算前一日 (D-1) 或 D-2 的预结算数据。
- **调度频率**: 每 30 分钟执行一次 (类比负荷聚合任务)。
- **核心逻辑 (`SettlementScheduler`)**:
    1.  **确定目标日期**: 默认为 `Today - 1` (D-1)。
    2.  **数据就绪检查 (Data Readiness Check)**:
        - **检查目标日期的数据是否存在**:
            - **负荷**: `unified_load_curve` (Target Date, MP_ONLY)
            - **电价**: `real_time_spot_price` & `day_ahead_price` (Target Date, 96/48 points)
            - **合同**: `contracts_aggregated_daily` (Target Date, 售电公司 & 全市场)
        - **检查当月数据**:
            - **机制电量**: `mechanism_energy_monthly` (Target Month)
    3.  **触发计算**:
        - 若数据全部就绪 且 (未计算 OR 需要更新)，则调用 `SettlementService.calculate_daily_settlement`。
        - **若数据不全**: 静默跳过 (Skip)，等待下一次调度。
