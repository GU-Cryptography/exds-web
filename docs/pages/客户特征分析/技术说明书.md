# 客户特征分析模块技术说明书

## 1. 模块概述
客户特征分析模块是电力交易辅助决策系统的核心组件之一，旨在通过对客户历史负荷数据的深度挖掘，自动识别客户的生产行为、经营趋势、用电模式及异常变动。该模块为电力交易员提供直观的客户画像，辅助制定差异化的交易建议和风险防控策略。

## 2. 业务逻辑架构
该模块采用基于**规则引擎**的标签生成系统，通过聚合多源数据构建分析上下文，运行预定义的业务规则库生成客户标签。

### 2.1 数据上下文 (Labeling Context)
分析过程首先构建一个丰富的上下文对象，包含以下数据源：
*   **短周期数据**：昨日 96 点负荷曲线。
*   **长周期数据**：过去 365 天的日用电量序列。
*   **典型负荷曲线**：过去 30 天负荷曲线的平均值（进行归一化处理）。
*   **时段电量信息**：基于分时电价规则计算的尖峰/高峰/平段/低谷/深谷电量占比。
*   **气象数据**：对应的气温、天气状况等。
*   **客户档案**：地理位置、行业分类等基础信息。

### 2.2 标签类别与判定规则
模块将标签分为以下八大类：

| 类别 | 描述 | 核心判定依据 (示例) |
| :--- | :--- | :--- |
| **生产班次 (Shift)** | 识别客户生产时间规律 | 匹配单班/双班/三班/连续生产等时间模板 (相似度 > 0.6) |
| **经营趋势 (Trend)** | 分析电量长期增减性 | 基于 STL 分解的趋势项斜率 (斜率 > 0.1%/日为扩张) |
| **日历特征 (Calendar)** | 识别周末及节假日行为 | 周末/工作日负荷比，春节/平时负荷比 |
| **季节气象 (Seasonal)** | 分析电量对温度的敏感性 | 1月/7月与春秋季均值对比，负荷与气温相关系数 |
| **稳定性 (Stability)** | 评估用电长期的波动性 | 日电量离散系数 (CV) |
| **行为规律 (Behavior)** | 评估日负荷曲线的复现性 | 近 30 天日曲线间的平均余弦相似度 |
| **能源设施 (Facility)** | 识别光伏、储能等设施使用 | 午间负荷凹陷识别，谷充峰放模式识别 |
| **成本偏好 (Cost)** | 评估客户对电价的响应程度 | 负荷曲线与电价曲线的相关系数（价格敏感度评分） |

### 2.3 指标体系
除了定性的标签，系统还维护一套定量的指标体系：
*   **质量评级 (A/B/C/D)**：结合规律性评分和波动率进行的综合评价。
*   **规律性评分 (0-100)**：基于曲线相似度的量化得分。
*   **价格敏感度评分 (0-100)**：反映客户避峰填谷的主动性。

## 3. 技术实现方案

### 3.1 后端架构
*   **服务层 (`CharacteristicService`)**：核心逻辑实现，负责上下文构建、规则驱动、结果持久化及异动告警管理。
*   **规则层 (`rules/`)**：各规则独立实现为 Rule 类，继承统一接口 `evaluate(context)`，便于扩展。
*   **算法层 (`algorithms/`)**：
    *   `stl_decomposition`：用于经营趋势的长期分量提取。
    *   `clustering`：使用余弦相似度计算曲线模式的一致性。
    *   `statistics`：离散系数 (CV)、线性回归斜率计算等。
*   **API 层 (`v1_load_characteristics.py`)**：提供总览、分布、变化、散点图及详情查询接口。

### 3.2 数据库方案 (MongoDB)
*   **`customer_archives`**：存储客户最新的全量标签列表（含手动标签）。
*   **`customer_characteristics`**：存储最新的量化指标、统计数据及基准曲线。
*   **`analysis_history_log`**：记录每日分析的快照，支持溯源分析。
*   **`customer_anomaly_alerts`**：存储识别出的异动告警，支持人工确认和备注。

### 3.3 前端展示
*   **总览页面 (`OverviewPage`)**：
    *   使用 **Recharts** 构建散点图（规模-稳定性分布）和 KPI 仪表盘。
    *   提供异动告警的滚动看板，强调实时关注点。
*   **详情页面 (`DetailPage`)**：
    *   **特征雷达图**：五维度（稳定性、增长力、规律性、调节潜质、价格敏感）直观展示客户画像。
    *   **深度分析图表**：长周期趋势线（带线性回归拟合）和典型日曲线（带分时电价背景色）。
    *   **标签管理**：支持自动/人工标签的切换与管理。

## 4. 关键交互流程
1.  **自动分析**：每日凌晨由调度任务触发，对全体签约客户进行特征扫描。
2.  **异动触发**：当识别到 `category='anomaly'` 的标签时，自动写入告警表并推送至前端面板。
3.  **人工修正**：交易员可在详情页手动添加备注类标签，系统将自动合并展示，确保画像的权威性与完整性。
