# 批发月度结算功能方案

## 1. 背景与目标

当前系统已实现批发侧日清结算（`settlement_daily`）及月度汇总展示，但尚未实现交易中心发布口径的“月度批发结算文件”管理能力。  
本方案用于新增“批发月度结算”模块，覆盖月度文件导入、月台账查询、与日清聚合对账。

目标：

1. 支持导入交易中心发布的月度结算文件（`.xls/.xlsx`）。
2. 按月保存“本公司主体行”月结数据。
3. 提供与 `settlement_daily` 的 `PLATFORM_DAILY` 版本聚合对账。

## 2. 已确认需求（产品口径）

1. 月度文件导入：支持 `.xls/.xlsx`，自动识别两行表头，只导入“主体行”。
2. 不导入“合计”“<全市场合计>”行。
3. 每个月只有一个版本；新导入同月文件时提示“覆盖”。
4. 合计行由前端自动计算展示，不落库。
5. 月度对账仅与 `settlement_daily` 的 `PLATFORM_DAILY` 版本聚合对比。
6. 对账分析项重点支持：合同电费、日前偏差电费、实时偏差电费、偏差回收费等。
7. 月度批发仅存“本公司主体行”。
8. 清算退补三列仅展示，不参与对账核心计算。
9. 不做争议处理相关功能。

## 3. 数据来源与口径

### 3.1 月度文件来源

- 文件：交易中心次月发布月度结算文件（示例：`现货月度结算.xls`）。
- 表头结构：两行表头（第一行为分组，第二行为字段）。
- 数据行类型：
  - 主体行（导入）
  - 合计行（过滤）
  - 全市场合计行（过滤）

### 3.2 日清对账来源

- 集合：`settlement_daily`
- 过滤条件：
  - `operating_date` 在目标月内
  - `version = PLATFORM_DAILY`
- 聚合方式：按月求和/加权均价（详见第 7 节）。

## 4. 功能范围

### 4.1 导入管理

1. 上传 `.xls/.xlsx` 文件。
2. 自动识别两行表头并建立字段映射。
3. 自动识别并提取本公司主体行。
4. 若月份已存在记录，弹窗提示覆盖；确认后覆盖写入。

### 4.2 月度台账查询

1. 月份列表查询（倒序）。
2. 指定月份详情查询（单条主体记录）。
3. 前端生成“合计行”显示（仅展示，不保存）。
4. 按年度维度展示当年 1-12 月数据（一个月份一行）。

### 4.3 对账分析

1. 展示“月度文件值 vs PLATFORM_DAILY聚合值”。
2. 展示差值与差异率。
3. 默认展示关键项：合同电费、日前偏差电费、实时偏差电费、偏差回收费、电能量电费、结算电费等。
4. 清算退补三列只展示文件值，不纳入核心对账差异统计。

## 5. 非范围（本期不做）

1. 多版本月度管理（每月仅保留一条当前版本数据）。
2. 合计行/全市场合计行入库。
3. 基于月结数据反算日结明细。
4. 争议处理流程。

## 7. 对账字段映射设计（与 PLATFORM_DAILY）

### 7.1 月结字段与日清聚合字段映射

1. `contract_fee`（月结合同电费）  
   对比：`sum(settlement_daily.contract_fee)`
2. `day_ahead_deviation_fee`（日前偏差电费）  
   对比：`sum(settlement_daily.day_ahead_fee)`
3. `real_time_deviation_fee`（实时偏差电费）  
   对比：`sum(settlement_daily.real_time_fee)`
4. `deviation_recovery_fee`（偏差回收费）  
   对比：`sum(settlement_daily.deviation_recovery_fee)`
5. `energy_fee_total`（电能量电费）  
   对比：`sum(settlement_daily.total_energy_fee)`
6. `settlement_fee_total`（结算电费）  
   对比建议：  
   `sum(settlement_daily.total_energy_fee + settlement_daily.deviation_recovery_fee)`  
   说明：该口径与月结“资金余缺费用合计”不完全同源，页面需标注“近似口径，仅用于经营侧对比”。

### 7.2 均价对比口径

1. 月结电能量均价：`energy_fee_total / actual_monthly_volume`
2. 月结结算均价：`settlement_fee_total / actual_monthly_volume`
3. 日清聚合均价：
   - 电能量均价：`sum(total_energy_fee) / sum(real_time_volume)`
   - 结算均价：`sum(total_energy_fee + deviation_recovery_fee) / sum(real_time_volume)`

页面需明确：分母口径可能存在 `actual_monthly_volume` 与 `sum(real_time_volume)` 的偏差。

## 8. 导入解析规则

### 8.1 表头识别

1. 读取前两行作为表头。
2. 第一行为空单元格采用向前填充分组名。
3. 第二行为空时使用分组名作为字段名。

### 8.2 主体行筛选

1. 排除名称为“合计”的行。
2. 排除名称包含“全市场合计”的行。
3. 仅保留本公司主体行（按“市场主体名称”匹配）。

主体名称配置建议：

- 方案 A（推荐）：后端配置常量 `WHOLESALE_SUBJECT_NAME`。
- 方案 B：从系统参数表读取，支持后台可配。

### 8.3 覆盖导入

1. 导入前先根据 `month` 查询是否存在。
2. 已存在返回 `409` + `requires_confirm=true`。
3. 前端二次确认后，带 `overwrite=true` 重试导入。

## 9. API 设计（建议）

统一前缀建议：`/api/v1/wholesale-monthly-settlement`

1. `GET /months`
   - 返回月份列表、导入时间、导入人。

2. `GET /{month}`
   - 返回该月主体结算记录。

3. `POST /import`
   - 参数：`file`、`overwrite`（默认 false）。
   - 行为：解析+校验+写入。

4. `GET /{month}/reconciliation`
   - 返回该月月结值、日清聚合值、差值、差异率明细。
   - 固定使用 `PLATFORM_DAILY`。

5. `DELETE /{month}`（可选）
   - 仅管理员使用，删除月记录。

## 10. 前端页面设计（建议）

新增页面：`批发月度结算`

采用单页三面板布局（不使用 Tab）：

1. 第一面板：操作区
   - 左侧：年度下拉框（选择年份，如 2026）。
   - 右侧：导入按钮（支持覆盖提示）。
   - 导入后自动刷新当前年度数据。

2. 第二面板：当年月度结算台账
   - 展示所选年度 1-12 月月度结算数据。
   - 一个月份一行，缺失月份显示为空或 `--`。
   - 底部展示前端自动计算“合计行”（不落库）。
   - 支持选中某个月份行（单选），作为第三面板的数据驱动条件。

3. 第三面板：所选月份对账区
   - 根据第二面板当前选中月份，展示该月份“月结 vs PLATFORM_DAILY 聚合”对账数据。
   - 展示字段：合同电费、日前偏差电费、实时偏差电费、偏差回收费、电能量电费、结算电费等。
   - 展示差值、差异率并高亮异常项。
   - 展示口径说明（尤其结算电费、均价分母差异）。

## 11. 校验规则

导入后端校验：

1. 文件月份必填且格式为 `YYYY-MM`。
2. 主体行必须且仅允许 1 条（按本公司名匹配）。
3. 关键金额字段可为空但需可转数值（空值按 `null` 存储，不强制转 0）。
4. 自动校验公式一致性（允许误差阈值）：
   - `energy_fee_total` vs 合同/日前/实时/绿电/调平求和
   - `settlement_fee_total` vs `energy_fee_total + fund_surplus_deficit_total`

1. 后端

   - 实现导入与对账展示
3. 联调与验收
   - 使用 `2026-01` 文件做导入与对账验证
   - 校验覆盖导入、差异计算

## 13. 验收标准

1. 可成功导入 `.xls/.xlsx`，且仅落本公司主体行。
2. 同月重复导入必须提示覆盖，确认后可覆盖。
3. 合计行只在前端展示，不出现在数据库记录。
4. 对账接口仅使用 `PLATFORM_DAILY` 聚合，结果可追溯。
5. 清算退补三列仅展示，不参与核心对账指标计算。
