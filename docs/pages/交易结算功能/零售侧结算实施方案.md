# 零售侧结算实施方案 (修订版)

## 一、业务背景

零售侧结算目标：根据每个客户的零售合同中绑定的套餐定价模型，结合每日峰谷时段和用户 48 时段电量，计算用户侧的零售电费。

结算模式分两种场景：
- **日常预结算**：每日计算，使用当日实时数据（用于展示当日预结算均价和电费）。执行“日均价封顶”保护。
- **月度正式结算**：次月生成，使用全月汇总数据（用于生成最终结算单）。执行“月结算均价封顶”保护。

2026年共 41 个有效合同，涉及 16 种套餐，主要分为以下三大类：

#### 类别B：上限分成类（5个合同）

## 三、核心业务规则

### 3.1 定价公式

1.  **分时展开**：按 **1.8:1.6:1:0.4:0.3** 比例将参考基准值展开为 5 个时段。
2.  **扣减计算**：`价格[时段] = 参考价[时段] - 约定价差 × 分成比例 + 浮动价`。
3.  **比例调节**：执行 463 号文件规定的比例保护逻辑。

### 3.2 价格比例调节规则 (463号文)

标准比例：**尖峰1.8 : 高峰1.6 : 平段1 : 低谷0.4 : 深谷0.3**。
- **锚点**：以平段价为基准。
- **下限调节**：尖峰/高峰价格若低于标准比例，则强制上调至标准值。
- **上限调节**：低谷/深谷价格若高于标准比例，则强制下调至标准值。

### 3.3 零售用户结算均价封顶执行规则 (细则 4.0)

根据《江西省电力零售市场实施细则（试行 4.0 修订版）》，系统需对零售用户执行严格的均价封顶保护。为平滑用户电费波动，避免超封顶金额集中累计到月底，本系统采取**“日清日结预演、月终统一清算”**的双重执行策略：

1.  **封顶价格 (CapPrice) 定义**：
    - **计算基准**：**当月市场化用户中长期交易平段合同加权平均价**。
    - **上浮比例**：
        - **非尖峰月份**：上浮不超过 **10%**（$CapPrice = 基准价 \times 1.10$）。
        - **尖峰月份**：上浮不超过 **15%**（$CapPrice = 基准价 \times 1.15$）。

2.  **双重结算封顶逻辑**：

    - **策略一：日常预结算（当前执行）**：
        - **目标**：每日计算当日电费，执行**“日均价封顶”**。
        - **逻辑**：当日名义均价若超过 $CapPrice$，则当日直接按 $CapPrice$ 结算。
        - **意义**：模拟月度封顶效果，让用户每日看到的累计电费更接近最终账单，避免超封顶金额累计到月底造成电费突增。

    - **策略二：月度正式结算（待开发）**：
        - **目标**：次月基于全月汇总数据，执行**“月结算均价封顶”**。
        - **现状**：**当前暂不实现**，待后续版本开发。
        - **逻辑（规划中）**：计算全月名义加权均价，若超过 $CapPrice$，则对全月各时段价格进行统一下调。

3.  **封顶压降执行策略**：
    - 为保持分时电价信号的形状不变，系统采用**同比例压降法**修正各时段单价：
      $$修正系数 k = \frac{CapPrice}{P_{nominal}}$$
      $$最终分时电价 = 原始分时电价 \times k$$
    - 此方法确保最终的加权均价严格等于 $CapPrice$，同时保留峰谷价差比例。

### 3.4 日常预结算流程（每日执行）

1.  **数据就绪检查**：确认当日 48 段电量、峰谷时段定义、现货/中长期基准价格到位。
2.  **合同规则解析**：识别用户生效合同、套餐参数及参考价基准。
3.  **初始分时定价**：根据定价模型和 463 号文比例调节计算 5 个时段的初始结算单价。
4.  **日均价封顶校验**：计算**当日**名义加权均价，若超过 `CapPrice`，执行同比例压降至 `CapPrice`。
5.  **费用核算**：将 48 段电量与修正后的时段单价相乘，汇总得出日总电费（预估值）。
6.  **聚合存储**：存储结果到 `retail_settlement_daily` 集合。

### 3.5 月度正式结算流程（待开发）

> **注：本阶段暂不实现月度正式结算功能，以下为远期规划流程。**

1.  **全月数据汇总**：聚合用户当月每日 48 点电量，汇总为全月分时电量。
2.  **基准价格确认**：获取官方发布的当月最终中长期平段合同加权均价，确立最终 `CapPrice`。
3.  **月度名义均价计算**：基于全月累计电量和各时段合同单价，计算全月名义结算均价。
4.  **月结算均价封顶执行**：
    - 若 `全月名义均价 > CapPrice`：计算统一修正系数 `k = CapPrice / 全月名义均价`，对所有时段价格进行压降。
    - 均价未超标：则不作调整。
5.  **最终账单生成**：计算最终电费，生成月度结算单并存档。

## 四、数据结构 (retail_settlement_daily)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `customer_id` | String | 客户唯一标识 |
| `date` | String | 结算日期 (YYYY-MM-DD) |
| `final_prices` | Object | 封顶修正后的 5 时段结算价 |
| `avg_price` | Number | **最终结算均价** (元/kWh) |
| `nominal_avg_price`| Number | 封顶前的名义均价 (用于回溯) |
| `is_capped` | Boolean | 是否触发了封顶保护 |
| `cap_price` | Number | 计算所依据的封顶价基准 |
| `total_load_mwh` | Number | 当日总电量 (MWh) |
| `total_fee` | Number | 当日总电费 (元) |
