
# 售电公司客户负荷总览页面设计方案 (V1.0)

**文档版本**：V3.0 (Time-Backtracking Edition)
**模块定位**：经营资产管理 / 电量业绩复盘 / 签约风险校验
**核心逻辑**：基于“全局时间回溯引擎”，支持从“单月”和“年累计”双维度对客户实测数据进行经营分析，重点解决签约估算偏差校验与历史业绩复盘需求。

---

## 1. 页面整体布局结构

页面采用 **“T型布局”**：顶部为全局控制栏，下方分为上（KPI）、中（透视）、下（明细）三层结构。

### 1.1 全局时间回溯引擎 (Global Time & Scope Engine)

位于页面最顶部（导航栏下方），控制全页面所有数据的统计口径。

* **UI 构成**：
* **日期切换器**：`[ < ] [ 2026年 03月 ▼ ] [ > ]`
* 仅精确到**月份**。
* 支持左右箭头快速切换月份。


* **统计口径开关**：`(●) 月度视图` | `(○) 年累计视图`
* 采用单选按钮组（Radio Group）或 胶囊切换器（Segmented Control）。




* **交互逻辑（核心规则）**：
* **场景 A：选择 [3月] + [月度视图]**
* 全页数据范围：3月1日 00:00 - 3月31日 24:00。
* 对比基准：去年3月（同比）。


* **场景 B：选择 [3月] + [年累计视图]**
* 全页数据范围：1月1日 00:00 - 3月31日 24:00。
* 对比基准：去年1月-3月同期累计（同比）。

## 2. 第一区域：动态 KPI 指标卡片 (Dynamic Stats)

**设计目标**：一览经营盘子大小与当前业绩好坏。
**布局建议**：一行 5 张卡片，背景轻微差异化（静态区 vs 动态区）。

| 卡片序号 | 标题 (动态) | 主数值展示 | 副指标/趋势 | 逻辑说明 |
| --- | --- | --- | --- | --- |
| **1 (静态)** | **签约客户** | **36** 户 |  | **静态参考**。显示当前有效的总户数。不受累计开关影响。 |
| **2 (静态)** | **签约规模** | ***12.2** 亿 kWh | 增幅: **[+百分比]%** | **静态参考**。显示当前有效的总签约电量，计算与去年总用电量的增幅比例。不受累计开关影响。 |
| **3 (动态)** | **当前总电量** | **[数值]** MWh | 增幅: **[+百分比]%** | **随时间引擎变化**。月度模式显示当月值；累计模式显示1-N月总和。 |
| **4 (动态)** | **综合峰谷比** | **[数值]** (如 2.8) | 尖峰平谷深谷电量占比 | **负荷质量指标**。累计模式下显示加权平均值，反映全盘成本属性。 |

## 3. 第二区域：资产结构透视 (Asset Intelligence)

**设计目标**：回答“电量是谁用的”、“增长来自哪里”、“谁是用电大户”。
**布局建议**：三栏等宽布局，高度约 280px - 320px。

### 左栏：客户电量贡献构成 (Concentration)

* **标题**：`{所选时间范围} 电量贡献构成` (例如 "1-3月累计电量构成")
* **图表类型**：**嵌套环形图 (Donut Chart)**
* **内环**：客户分级占比（如：战略客户 / 重点客户 / 普通客户）。
* **外环**：Top 5 大客户独立扇区 + "其他"扇区。


* **交互**：悬停显示具体电量值及占比%。

### 中栏：业绩涨跌龙虎榜 (Growth Ranking)

* **标题**：`{所选时间范围} 电量同比增减 Top 5`
* **图表类型**：**正负条形图 (Bar Chart)** 或 **双列列表**
* **左半部分 (红榜)**：**增量**最大的前5名客户。显示：`客户名` + `+200 MWh`。
* **右半部分 (绿榜)**：**减量**最大的前5名客户。显示：`客户名` + `-150 MWh`。


* **价值**：累计模式下，能一眼看出今年谁在扩产，谁在停产。

### 右栏：负荷效率极值榜 (Efficiency Extremes)

* **标题**：`{所选时间范围} 峰谷比极值 Top 5`
* **图表类型**：**分组列表**
* **组 A (高成本/尖峰型)**：峰谷比最高的 5 个客户。标记为 ⚡️ (高关注)。
* **组 B (优质/平稳型)**：峰谷比最低的 5 个客户。标记为 🔋 (优质)。


* **数据源**：累计模式下取加权平均峰谷比，消除单月偶然波动。

## 5. UI 视觉设计建议 (给设计师的 Note)

1. **色彩规范**：
* **涨跌色**：建议遵循金融/电力行业习惯（通常红涨绿跌，或根据客户VI定义）。
* **时段色**：必须保持全系统一致。
* **尖 (Sharp)**：深红 / 紫色 (警示感)
* **峰 (Peak)**：橙色
* **平 (Flat)**：绿色 / 青色
* **谷 (Valley)**：蓝色 / 浅蓝




2. **空状态处理**：
* 如果某个月份（如未来月份）没有实测数据，KPI 显示 "--"，图表显示 "暂无数据" 插画。


3. **累计视图的视觉暗示**：
* 当用户切换到 `(○) 年累计视图` 时，建议 KPI 卡片区域的背景色或边框出现微妙变化（例如增加一圈淡蓝色光晕），以提醒用户当前看的是“大盘子”数据。

## 6. 开发逻辑边界 (给开发者的 Note)

* **数据懒加载**：
* 初始化时只加载“本月”+“月度视图”数据。
* 切换到“累计视图”时，触发 API 重新拉取聚合数据（不要在前端进行 12 个月的加法计算，数据量大容易卡顿，应由后端聚合）。


* **去年全量 (Last Year Total)**：
* 这是指“去年1月1日-12月31日”的完整值，是一个定值（Constant），不需要随月份选择器变化。


* **同比计算规则**：
* 如果去年同期数据缺失（新户），同比显示 "新户" 或 "--"。
* 如果分母为 0，处理除零异常。


* **月份选择范围**：
只能选择2026年1月-2026年12月
