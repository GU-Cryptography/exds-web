# 负荷特征分析服务技术文档

## 1. 概述

负荷特征分析服务 (`CharacteristicService`) 是一个基于规则引擎的客户用电特征自动识别系统，通过分析历史负荷数据为每个客户生成标签画像。

### 1.1 核心能力

- **长周期分析**：识别经营趋势、季节性、气温敏感性
- **短周期分析**：识别生产班次、行为规律、用电模式
- **异动检测**：识别形状异动、力度异动、规律偏离等异常

### 1.2 架构

```
webapp/services/characteristics/
├── service.py             # 核心服务入口 (CharacteristicService)
├── constants.py           # 阈值配置
├── engine/
│   ├── base_rule.py       # 规则基类
│   └── context.py         # 数据上下文 (LabelingContext)
├── rules/                 # 规则库
│   ├── long_term.py       # 长周期规则
│   ├── short_term_*.py    # 短周期规则
│   ├── anomaly.py         # 异动规则
│   └── calendar.py        # 日历规则
└── algorithms/            # 算法库
    ├── statistics.py      # CV, 零值计数
    ├── stl_decomposition.py # STL 分解
    └── clustering.py      # 余弦相似度, 模板匹配
```

## 2. 规则引擎

### 2.1 规则基类

所有规则继承 `BaseRule`，实现以下接口：

| 属性/方法 | 说明 |
|-----------|------|
| `rule_id` | 规则唯一标识 |
| `category` | 规则类别: `long_term` / `short_term` / `anomaly` |
| `evaluate(context)` | 输入上下文，返回 `Tag` 或 `None` |
| `create_tag(name, confidence, reason)` | 创建标签辅助方法 |

### 2.2 数据上下文 (LabelingContext)

规则评估的输入数据结构：

| 字段 | 类型 | 说明 |
|------|------|------|
| `customer_id` | str | 客户ID |
| `date` | datetime | 分析日期 |
| `load_series` | List[float] | 当日96点负荷 |
| `total_load` | float | 当日用电量 |
| `typical_load_series` | List[float] | 30天典型曲线 |
| `long_term_dates` | List[str] | 长周期日期序列 |
| `long_term_values` | List[float] | 长周期日电量序列 |
| `load_rate` | float | 负荷率 (avg/max) |
| `peak_valley_diff` | float | 峰谷差 |
| `weather_data` | Dict | 天气数据 |
| `weather_service` | Any | 天气服务依赖注入 |

## 3. 规则清单

### 3.1 长周期规则 (long_term.py)

| 规则ID | 标签名 | 判定逻辑 |
|--------|--------|----------|
| `rule_production_trend` | 产能扩张 / 产能萎缩 / 经营稳健 | STL趋势项斜率 > ±0.1% |
| `rule_stability` | 经营稳健 / 波动剧烈 / 间歇停工 | CV < 0.2 且 零值 < 20% |
| `rule_seasonal` | 冬夏双峰型 | 冬夏月电量峰值 vs 春秋谷值 |
| `rule_temperature_sensitive` | 气温敏感型 / 气温钝化型 | 日电量与气温相关性 |

### 3.2 短周期规则 (short_term_*.py)

| 规则ID | 文件 | 标签名 | 判定逻辑 |
|--------|------|--------|----------|
| `shift_pattern_analysis` | short_term_shift.py | 连续生产 / 单班 / 双班 / 三班 | 负荷率 > 0.6 或模板匹配 |
| `rule_short_behavior_01` | short_term_behavior.py | 机器规律型 / 随机波动型 | 负荷率+CV组合判定 |
| `rule_short_asset_01` | short_term_asset.py | 分布式光伏 / 分布式储能 | 日间发电特征 |
| `rule_short_cost_01` | short_term_cost.py | 移谷填峰 / 避峰用电 | 分时电量占比 |

### 3.3 异动规则 (anomaly.py)

| 规则ID | 标签名 | 判定逻辑 | 严重程度 |
|--------|--------|----------|:--------:|
| `rule_anomaly_shape` | 形状异动 ⚡ | 当日曲线与典型曲线相似度 < 0.7 | warning |
| `rule_anomaly_peak` | 峰值异动 📈 | 最大负荷偏离历史均值 > 2σ | warning |
| `rule_anomaly_regularity` | 规律异动 📊 | 近5天波动率 > 历史2.5倍 | warning |
| `rule_anomaly_magnitude` | 力度异动 💪 | 日电量偏离30天均值 > 50% | warning |
| `rule_anomaly_extreme` | 剧烈异动 🔴 | 日电量偏离 > 80% 且 > 20 MWh | critical |
| `rule_anomaly_day_surge` | 日环比突变 🔥 | 日环比变化 > 100% 且 > 20 MWh | critical |

## 4. API 接口

### 4.1 接口列表

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/load-characteristics/analyze/{customer_id}` | 单客户分析 |
| POST | `/load-characteristics/analyze/batch/all` | 全量批量分析 |
| GET | `/load-characteristics/overview` | 总览统计 |
| GET | `/load-characteristics/customers` | 客户列表 |
| GET | `/load-characteristics/customer/{id}` | 客户详情 |
| GET | `/load-characteristics/customer/{id}/daily-trend` | 日电量趋势 |
| GET | `/load-characteristics/customer/{id}/monthly-energy` | 月电量统计 |

### 4.2 接口示例

#### 手动触发分析
```http
POST /api/v1/load-characteristics/analyze/690f039915fdeb957cdd228f?date=2026-01-29
```

响应:
```json
{
  "status": "success",
  "customer_id": "690f039915fdeb957cdd228f",
  "tags_generated": 5
}
```

#### 获取总览
```http
GET /api/v1/load-characteristics/overview
```

响应:
```json
{
  "kpi": {

    "total_customers": 47,
    "anomaly_count_today": 3,
    "avg_regularity_score": 72.5
```

## 5. 数据集结构

### 5.1 customer_characteristics

存储客户特征分析结果。

| 字段 | 类型 | 说明 |
|------|------|------|
| `customer_id` | String | 客户ID (主键) |
| `customer_name` | String | 客户名称 |
| `updated_at` | ISODate | 最后更新时间 |
| `long_term` | Object | 长周期指标 |
| `short_term` | Object | 短周期指标 |
| `regularity_score` | Number | 规律性评分 (0-100) |
| `quality_rating` | String | 质量评级 (A/B/C) |
| `baseline_curve` | Array | 典型负荷曲线 (96点) |

### 5.2 customer_anomaly_alerts

存储异动告警历史。

| 字段 | 类型 | 说明 |
|------|------|------|
| `customer_id` | String | 客户ID (复合主键) |
| `alert_date` | String | 告警日期 (复合主键) |
| `alert_type` | String | 告警类型 (复合主键) |
| `severity` | String | 严重程度: low/warning/critical |
| `confidence` | Number | 置信度 (0-1) |
| `reason` | String | 告警原因 |
| `metrics` | Object | 指标快照 |
| `acknowledged` | Boolean | 是否已确认 |

### 5.3 analysis_history_log

存储分析历史日志。

| 字段 | 类型 | 说明 |
|------|------|------|
| `customer_id` | String | 客户ID |
| `date` | String | 分析日期 |
| `tags` | Array | 生成的标签列表 |
| `analyzed_at` | ISODate | 分析时间 |

## 6. 标签存储

所有生成的标签统一存储在 `customer_archives.tags` 字段：

```json
{
  "customer_id": "...",
  "tags": [
    {
      "name": "连续生产",
      "source": "AUTO",
      "confidence": 0.85,
      "rule_id": "shift_pattern_analysis",
      "reason": "日均负荷率 0.72 高",
      "created_at": "2026-01-29T10:00:00Z"
    }
  ]
}
```

**标签保留策略**:
- `source=MANUAL`: 人工标签永久保留
- `source=AUTO`: 每次分析时清空并重新生成

## 8. 阈值配置

文件: `constants.py`

| 常量 | 默认值 | 说明 |
|------|:------:|------|
| `TREND_SLOPE_THRESHOLD` | 0.001 | 趋势斜率阈值 (~36%年增长) |
| `SEASONALITY_STRENGTH_THRESHOLD` | 0.4 | 季节性强度阈值 |
| `STABILITY_CV_THRESHOLD` | 0.2 | 稳定性-离散系数阈值 |
| `ZERO_COUNT_RATIO_THRESHOLD` | 0.2 | 停产-零值比例阈值 |
| `SHIFT_LOAD_RATE_THRESHOLD` | 0.6 | 连续生产-负荷率阈值 |
| `ANOMALY_CONFIDENCE_DEFAULT` | 0.8 | 异动默认置信度 |
