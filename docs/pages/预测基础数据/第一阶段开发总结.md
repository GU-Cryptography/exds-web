## 二、功能概述

- 数据可用性矩阵（按12:00是否存在数据判定）
  - GET /api/v1/forecast-base-data/availability
  - 参数：
    - base_date: YYYY-MM-DD（基准日期）
    - date_range: recent_3（默认）| recent_7 | historical_10
  - 返回：DataAvailabilityResponse（包含 base_date、date_range、15×N 可用性矩阵，sample_timestamp 为样本时间点12:00）

- 多曲线对比（96点）
  - POST /api/v1/forecast-base-data/curves
  - 请求体：[{ "data_item_id": 1..15, "date": "YYYY-MM-DD" }, ...]
  - 返回：MultipleCurvesResponse（每条曲线含 data_item_name、data[]、completeness、total_points）

## 三、关键技术实现

- 数据项映射（DATA_ITEM_CONFIG）
  - weekly_forecast×5：均取 value 字段，按 info_name 过滤（系统负荷预测、统调光伏、统调风电、统调水电(含抽蓄)、省间联络线容量）
  - daily_release×5：system_load_forecast、pv_forecast、wind_forecast、nonmarket_unit_forecast、tieline_plan
  - real_time_generation×4：total_generation、wind_generation、solar_generation、hydro_with_pumped_total_generation
  - real_time_tieline×1：total_tieline_plan

- 查询区间与“24:00”规范
  - 曲线查询区间严格采用左开右闭 (date 00:00, date+1 00:00]，确保次日00:00被包含
  - 将次日00:00统一格式化为字符串“24:00”，其余时间使用“%H:%M”

- 防御式时间点格式化（_format_time_point）
  - 对 point、datetime、目标字段进行严格校验，缺失/非数值则跳过该点（不以0填补）
  - completeness = 有效点数量 / 96 × 100（保留两位小数）

- 索引与性能
  - _ensure_indexes：若集合不存在任何含 datetime 的索引，尝试创建单字段 datetime 索引（捕获异常并记录日志，不阻断业务）
  - 查询尽量使用投影（仅取 datetime 与目标字段）

- 日志与错误处理（按规范）
  - Service 层抛出 ValueError，API 层转换为 HTTPException（400）
  - 其他异常记录日志（exc_info=True），统一返回 500
  - 路由提供 summary 与中文 description

## 五、已处理的潜在问题与风险

- 修复/规避 `_format_time_point` 可能因 point 为 None、缺 datetime 或缺目标字段而报错的风险；避免以 0 掩盖缺失导致 completeness 失真
- 可用性判定统一以 12:00 是否存在，避免“以任意时刻存在即为可用”的语义不一致
- 索引创建采取“非侵入式、仅在缺失时补充”的策略，避免与既有唯一索引冲突
