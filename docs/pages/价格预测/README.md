## 🗂️ 数据集说明

### 已有数据集（来源：`docs/pages/预测基础数据/dataset_structures.md`）

**特征数据**（用于模型输入）：
1. `weekly_forecast` - 周预测数据（负荷、风电、光伏、水电、联络线）
2. `daily_release` - 日前预测数据（系统负荷、光伏、风电、非市场化机组、联络线）
3. `real_time_generation` - 实时发电出力（各类型机组出力）
4. `real_time_tieline` - 实时联络线计划

**标签数据**（预测目标）：
1. `day_ahead_spot_price` - 日前现货价格（每日96点）
2. `real_time_spot_price` - 实时现货价格（每日96点）

**数据粒度**：15分钟，每日96个数据点

## ❓ 常见问题（FAQ）

### Q1: 如果历史数据不足6个月怎么办？

**答**：
1. 可以先用现有数据建立基线模型，但需要降低期望（准确率可能不理想）
2. 持续积累数据，每周/每月重新训练模型，逐步提升性能
3. 考虑引入外部数据源（如邻省价格、天气数据）来补充特征

### Q2: 模型评估指标达不到验收标准怎么办？

**答**：
1. 检查数据质量（缺失值、异常值）
2. 尝试不同的模型（XGBoost、LightGBM、LSTM）
3. 进行超参数调优（GridSearchCV）
4. 增加更多特征（天气、节假日、滞后特征）
5. 如果仍然不理想，与业务方沟通调整验收标准

### Q3: 如何处理模型性能退化？

**答**：
1. 启用监控服务（`model_monitoring_task.py`）定期检查
2. 如果检测到性能下降，执行重新训练（`retrain_model_weekly.sh`）
3. 对比新旧模型效果，选择性能更好的模型激活

### Q4: 前端开发时需要注意什么？

**答**：
1. **严格遵循前端开发规范**（`docs/spec/前端开发规范.md`）
2. **Material-UI Grid v7 语法**：必须使用 `size` 属性
3. **图表全屏**：必须使用 `useChartFullscreen` Hook
4. **Loading 状态管理**：区分首次加载和数据刷新，避免卸载组件导致全屏状态丢失
5. **响应式设计**：移动端优先，测试多种设备

### Q5: 后端开发时需要注意什么？

**答**：
1. **严格遵循后端开发规范**（`docs/spec/后端开发规范.md`）
2. **数据库访问**：必须通过 `webapp.tools.mongo.DATABASE` 进行
3. **时间处理**：使用本地时间（`datetime.now()`），处理24:00特殊情况
4. **数据模型**：使用 Pydantic 模型，定义 Create/Update/Full/ListItem/ListResponse 五种模型
5. **日志记录**：使用 `logging` 而非 `print`
