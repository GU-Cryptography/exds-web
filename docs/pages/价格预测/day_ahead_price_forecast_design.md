# 日前价格预测页面设计方案

## 1. 业务目标

为交易员和分析师提供日前电价预测结果的可视化分析工具，支持对模型表现进行全方位的回溯与评估。核心功能包括：
1.  **多版本回溯**：查看特定日期的所有预测版本（支持 D-1 不同模型版本）。
2.  **预测验证**：将预测曲线与实际出清曲线（日前现货价格）进行对比，包含置信区间展示。
3.  **深度评估**：基于 `forecast_accuracy_daily` 数据集，展示多维度的准确度指标（分时段准确率、统计特征等）。

## 2. 数据源分析

页面涉及以下核心数据集：

| 数据集 | 集合名称 | 用途 | 关键字段 |
| :--- | :--- | :--- | :--- |
| **预测数据** | `price_forecast_results` | 展示预测曲线 | `forecast_id`, `target_date`, `predicted_price`, `confidence_80_lower/upper` |
| **真实数据** | `day_ahead_spot_price` | 展示真实曲线 | `datetime`, `avg_clearing_price` |
| **准确度** | `forecast_accuracy_daily` | 展示评估指标 | `wmape_accuracy`, `period_accuracy`, `stats`, `rate_90_pass` |

## 3. 页面布局设计

页面采用 **"控制栏 - 主图表 - 评估详情"** 的垂直流式布局。

### 3.1 区域 A：控制栏 (Filters)
位于页面顶部 `Paper` 容器中。
-   **日期选择器**：选择 `target_date`（目标日）。
-   **预测版本选择**：下拉框，展示该日期下可用的所有 `forecast_id`。
    -   显示格式：`{created_at (HH:mm)} - {model_version}`
    -   逻辑：默认选中最新生成的版本。
    -   辅助信息：显示当前选中版本的 `model_type`。

### 3.2 区域 B：趋势对比图 (Chart)

核心可视化区域，高度 **450px**，支持全屏模式。
-   **X轴**：时间（00:15 - 24:00）。
-   **Y轴**：价格（元/MWh）。
-   **图层配置**：
    1.  **置信区间 (Area)**：浅蓝色半透明填充 (`confidence_80_lower` ~ `confidence_80_upper`)，仅在有此数据时显示。
    2.  **预测价格 (Line)**：蓝色实线 (`predicted_price`)，带交互点。
    3.  **实际价格 (Line)**：红色实线 (`avg_clearing_price`)，若当日未出清则不显示。
-   **交互**：Tooltip 同时显示预测值、实际值和两者差值 (Error)。

### 3.3 区域 C：准确度评估详情 (Accuracy Details)
仅当存在真实数据（即有评估记录）时显示。此区域分为三个子模块：

#### 3.3.1 核心指标概览 (KPI Cards)
第一行展示最重要的模型评分。
-   **WMAPE 准确率**: 核心指标，大字号展示。颜色编码（>90% 绿色, >85% 橙色, <85% 红色）。
-   **达标徽章**: 展示 `90%考核达标` (`rate_90_pass`) 和 `85%考核达标` (`rate_85_pass`) 的 Chip 状态。
-   **方向准确率**: 展示涨跌趋势预测的正确率。
-   **误差绝对值**: MAE (平均绝对误差) 和 RMSE (均方根误差)。
-   **拟合度**: R² Score。

#### 3.3.2 分时段准确度 (Period Breakdown)
展示不同负荷时段（根据分时电价规则动态生成）的预测表现。
-   展示形式：表格或条形图。
-   数据来源：`period_accuracy` 对象 (如 `{"高峰": 86.5, "平段": 82.3, "低谷": 91.2}`).
-   业务价值：帮助分析师判断模型在特定时段（如早晚高峰）的预测能力是否不足。

#### 3.3.3 当日数据特征 (Daily Statistics)
展示当日价格曲线的统计特征，辅助解释预测偏差的原因。
-   数据来源：`stats` 对象。
-   指标：最高价 (`max_value`)、最低价 (`min_value`)、均价 (`mean_value`)。
-   特殊标记：是否包含负电价 (`has_negative`) —— 负电价通常会导致预测难度剧增。

## 4. API 接口设计

### 4.1 获取预测版本列表
`GET /api/v1/price-forecast/versions`

**请求参数**:
-   `target_date`: YYYY-MM-DD
-   `forecast_type`: (可选) 默认 `d1_price`

**响应示例**:
```json
[
  {
    "forecast_id": "20251215_0920",
    "forecast_type": "d1_price",
    "model_version": "v1.0.3",
    "model_type": "d1_price_model",
    "created_at": "2025-12-14T09:20:00"
  }
]
```

### 4.2 获取图表数据 (合并视图)
`GET /api/v1/price-forecast/data`

**请求参数**:
-   `forecast_id`: 字符串 (必需)
-   `target_date`: YYYY-MM-DD

**响应结构**:
-   自动聚合 `price_forecast_results` 和 `day_ahead_spot_price`。
-   对于缺失的实际价格，字段值为 `null`。

### 4.3 获取评估详情
`GET /api/v1/price-forecast/accuracy`

**请求参数**:
-   `forecast_id`: 字符串

**响应结构**:
-   直接返回 `forecast_accuracy_daily` 中的文档。

## 6. 前端组件实现建议

1.  **分时段表现**:
    -   使用 `List` 或简单条形图展示 `period_accuracy`。
    -   Key 值（如"高峰"）直接来自后端 Map，无需前端硬编码，支持不同省份规则。

2.  **数据特征卡片**:
    -   如果 `has_negative` 为 `true`，显示醒目的警告图标，因为负电价对预测模型通常是巨大挑战。

3.  **颜色映射**:
    -   WMAPE > 90%: `success.main`
    -   WMAPE 85-90%: `warning.main`
    -   WMAPE < 85%: `error.main`

4.  **空状态处理**:
    -   当 `forecast_accuracy_daily` 为空（例如今日预测明日，明日真实价格尚未出），整个区域 C 应替换为 "等待实际价格出清后生成评估报告" 的提示或隐藏。
