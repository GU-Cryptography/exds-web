## 1. 核心评估指标定义

在系统各处（包括列表列、详情页头部、报表统计）统一采用以下标准定义：

### 1.1 完整率 (Integrity Rate)
*   **公式**: `1 - (断点天数 / (昨日 - 开始日期))`
*   **含义**: 衡量全生命周期内数据的**时间轴连续性**。
*   **断点定义**: 某日既无有效电表数据(Meter)，也无有效计量点聚合数据(MP)，即视为**完全断点**。

### 1.2 可靠率 (Reliability Rate)
*   **公式**: `(计量点组合完整的天数 / 总天数) * 100%`
*   **含义**: 衡量**计量点 (MP) 源数据**的完整程度。
*   **判定标准**:
    *   系统依据档案检查每日应采的计量点数量 (Expected Count)。
    *   若某日 **实际采集MP数量 < 档案应采数量** (例如应采10个，实采9个)，则该日数据判为**不可靠 (Unreliable)**。

### 1.3 准确率 (Accuracy Rate)
*   **公式**: `1 - AVG( |(Meter电量 - MP电量) / Meter电量| )`
*   **统计范围**: 仅计算**双边数据重叠**（即当日既有完整Meter数据，也有完整MP数据）的日期。
*   **含义**: 衡量电表总表数据与计量点聚合数据之间的**一致性**。

### 3.2 功能详解

#### A. 工具栏交互 (Toolbar Actions)
> **位置**: 位于列表上方右侧 (Material UI TableToolbar 标准位置)。

1.  **[📂 导入数据...]**:
    *   点击打开 **“导入数据弹窗”** (见下文详情)。
2.  **[⚡ 执行数据聚合]**:
    *   **徽标**: 若有待聚合数据，显示红色徽标或后缀 `(125)`。
    *   点击打开 **“聚合选项弹窗”** (不直接执行)。

#### B. 聚合选项弹窗 (Aggregation Modal)
> **设计目标**: 在一个弹窗内清晰区分“日常操作”和“高级操作”。

*   **标题**: "执行负荷数据聚合"
*   **选项 (Radio Group)**:
    *   **(o) 增量聚合 (推荐)**:
        *   说明: "仅计算系统中已导入但未聚合的新数据 (共 125 天次)。速度快，不影响历史数据。"
    *   **( ) 重算历史数据**:
        *   说明: "强制重新计算指定时间段内的所有数据。适用于修正系数或修复历史错误。"
        *   **[关联控件]**: 当选中此项时，下方显示 `[ 开始日期 ] ~ [ 结束日期 ]` 选择器。
*   **底部按钮**: `[取消]`, `[开始执行]` (点击后进入进度模态)。

#### C. 聚合进度反馈 (Progress Modal)
> **场景**: 点击“开始执行”后触发。

```text
+-----------------------------------------------------------------------+
|  正在执行数据聚合...                                                   |
|                                                                       |
|  [===========================>           ]  68%                       |
|                                                                       |
|  正在处理: 江西A工厂 (2025-06-12)                                      |
|  已完成: 85 / 125 天次                                                 |
+-----------------------------------------------------------------------+
```
*   **包含**: 进度条、百分比、当前处理对象动态文本。
*   **完成态**: 显示成功/失败统计，按钮变为“关闭”。

#### D. [复用] 导入数据弹窗设计 (同 v2.0)
*   **Tabs**: 电表 / 计量点。
*   **上传**: 拖拽批量上传。
*   **反馈**: 结果统计 Chips + 错误列表 + 智能缺失预警。

## 5. UI 视觉规范

*   **状态色谱**:
    *   **Pending (蓝 #1976d2)**: 已上传但未聚合。
    *   **Health (绿 #2e7d32)**: 数据完整且准确。
    *   **Warning (橙 #ed6c02)**: 可靠率不足（缺MP）或 误差轻微（1-3%）。
    *   **Critical (红 #d32f2f)**: 完全断点 或 误差严重（>3%）。
    *   **Neutral (灰 #9e9e9e)**: 无数据周期或未签约时段。

*   **响应式适配**:
    *   **Desktop**: 完整 3行布局。
    *   **Mobile**: 标题显示，第一行KPI可横向滑动，第二行筛选折叠为“筛选”图标，第三行列表转卡片。

## 4.3 系数校核弹窗 (Coefficient Calibration Dialog) 详细设计

点击工作台中的 `[ ⚡ 计算推荐系数 ]` 按钮触发此弹窗。

### A. 弹窗布局 (ASCII Map)

```text
+---------------------------------------------------------------+
|                                                               |
|  1. 分析配置 (Configuration)                                  |
|  算法模型: [ 约束最小二乘法 (Constrained Least Squares) v ]   |
|  样本策略: [ 单日 (推荐) v ]     (2025-01-24)                 |
|  有效样本: 1 天 (48个点)                                      |
|                                                               |
|  2. 拟合分析 (Fitting Analysis)                               |
|  +---------------------------------------------------------+  |
|  |  拟合置信度: 🟢 高 (High)   残差(Residual): 0.45%       |  |
|  +---------------------------------------------------------+  |

|  |  101,500 kWh             |  102,000 kWh                 |  |
|  +--------------------------+------------------------------+  |

|  -----------------------------------------------------------  |
|  目标电表          当前系数        >>>  🔥 推荐系数           |
|  Meter-001        1.00                 0.9850               |
|  Meter-002        1.00                 0.9520               |
|  -----------------------------------------------------------  |
|                                                               |

|                                                               |
|  [ 取消 ]                                     [ 应用配置 ]    |
+---------------------------------------------------------------+
```

### B. 功能逻辑说明

1.  **核心算法 (Algorithm)**:
    *   采用 **约束最小二乘法 (Constrained Least Squares)**。
    *   目标函数: `min Σ(MP_t - Σ(Meter_i_t * k_i))^2`。
    *   约束条件: `0 <= k_i <= 1` (非负且不超过100%)。

2.  **样本选取 (Sampling)**:
    *   **单日 (推荐)**: 仅取最近的1个"完整日"作为样本 (48个点)，响应最快。
    *   **连续7天**: 取最近的连续7个"完整日"作为样本，平衡稳定性。
    *   **近30天**: 样本量更大，适合波动较大的场景。

3.  **结果评估**:
    *   **残差 (Residual)**: 拟合后的总偏差率。若残差 > 1%，提示"拟合效果不佳，建议检查电表接线或CT变比"。
    *   **置信度**: 基于样本数量和残差评估 (High/Medium/Low)。

4.  **应用范围**:
    *   **更新档案**: 修改 `customer_archives` 中对应电表的 `allocation_percentage`。
    *   **重算历史**: 触发重算任务，使用新系数修正历史负荷数据。
