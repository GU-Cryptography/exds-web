#### B. [新] 聚合进度反馈设计 (Progress Feedback)
> **设计原则**: 聚合操作（尤其是涉及几百天次的历史重算）可能耗时 10秒~几分钟。**必须**给用户明确的进度反馈，不能只有一个转圈的 Loading。

1.  **增量聚合 (Incremental)**:
    *   **触发**: 点击主按钮 `[⚡ 执行增量聚合]`。
    *   **交互**:
        *   按钮进入 `Loading` 状态 (不可点击)。
        *   若预估时间 < 3秒，仅显示按钮内转圈。
        *   若待处理数据量大 (>100条)，**弹出进度对话框** (同下方重算设计)。

2.  **历史重算 (Recalculate)**:
    *   **触发**: 在“重算弹窗”点击 `[开始重算]` 后。
    *   **交互**: 立即关闭配置弹窗，**打开“任务执行中”模态窗口** (Modal, 禁止点击背景关闭)。

#### C. 任务进度模态窗口 (Task Progress Modal) ASCII 图

```text
+-----------------------------------------------------------------------+
|  正在执行数据聚合...                                                   |
|                                                                       |
|  [===========================>           ]  68%                       |
|                                                                       |
|  正在处理: 江西A工厂 (2025-06-12)                                      |
|  已完成: 85 / 125 天次                                                 |
|                                                                       |
|  [ ⛔ 停止任务 ] (仅在极端耗时显示)                                     |
+-----------------------------------------------------------------------+
```

*   **UI 元素**:
    *   **进度条 (LinearProgress)**: 确定的百分比进度。
    *   **百分比数字**: 比如 `68%`。
    *   **当前活动 (Status Text)**: 动态显示正在计算哪个客户/哪一天，让界面“活”起来。
    *   **计数器**: `85 / 125`。
*   **完成态**:
    *   进度条跑满 100%。
    *   图标变为 ✅ 绿色。
    *   文字提示: "聚合完成，成功 120 条，失败 5 条"。
    *   按钮变为 `[关闭]` 或 `[查看日志]`。
