## 1. 核心评估指标定义

在系统各处（包括列表列、详情页头部、报表统计）统一采用以下标准定义：

### 1.1 完整率 (Integrity Rate)
*   **公式**: `1 - (断点天数 / (昨日 - 开始日期))`
*   **含义**: 衡量全生命周期内数据的**时间轴连续性**。
*   **断点定义**: 某日既无有效电表数据(Meter)，也无有效计量点聚合数据(MP)，即视为**完全断点**。

### 1.2 可靠率 (Reliability Rate)
*   **公式**: `(计量点组合完整的天数 / 总天数) * 100%`
*   **含义**: 衡量**计量点 (MP) 源数据**的完整程度。
*   **判定标准**:
    *   系统依据档案检查每日应采的计量点数量 (Expected Count)。
    *   若某日 **实际采集MP数量 < 档案应采数量** (例如应采10个，实采9个)，则该日数据判为**不可靠 (Unreliable)**。

### 1.3 准确率 (Accuracy Rate)
*   **公式**: `1 - AVG( |(Meter电量 - MP电量) / Meter电量| )`
*   **统计范围**: 仅计算**双边数据重叠**（即当日既有完整Meter数据，也有完整MP数据）的日期。
*   **含义**: 衡量电表总表数据与计量点聚合数据之间的**一致性**。

## 2. 页面与功能架构

模块整体采用 **“监控仪表盘 (Dashboard) + 诊断工作台 (Workbench)”** 的主从架构，并将数据维护操作（导入/聚合）深度集成于诊断流程中。

### 2.1 核心业务闭环
**“发现 (Discover) -> 修复 (Fix) -> 验证 (Verify)”**

### 3.2 功能详解

#### A. [新] 聚合操作交互 (Split Button)
> **设计目标**: 区分“日常维护”与“历史修正”场景。此前单一的按钮无法满足“只算这一周”或者“强制重算上个月”的需求。

1.  **主按钮 (Primary Action)**: **`[ ⚡ 执行增量聚合 ]`**
    *   **定义**: 仅针对系统中状态为 `Pending` (有Raw Data但无Unified Load) 的数据进行计算。
    *   **徽标提示**: 若系统检测到有待聚合数据，按钮上显示徽标或后缀 `(125天次)`，提示用户点击。
    *   **场景**: 日常导入数据后，用户习惯性点击此按钮，快速完成更新，速度最快。

2.  **下拉菜单 (Dropdown Menu)**:
    *   点击主按钮右侧的箭头 `[ v ]` 触发。
    *   **选项**: **`[ 🔄 重算历史数据... ]`**

3.  **重算弹窗 (Re-aggregation Dialog)**:
    *   **标题**: "重算历史负荷数据"
    *   **时间范围选择**:
        *   `[ 开始日期 ]` ~ `[ 结束日期 ]` (必填)
        *   提供快捷选项: `[近7天]`, `[近30天]`, `[本月]`, `[上月]`。
    *   **范围说明 (Scope)**:
        *   默认: **所有客户** (因为这是 Dashboard 级的全局操作)。
    *   **警告信息 (Warning)**:
        *   "⚠️ **注意**：此操作将强制重新计算所选时间段内的所有负荷数据。如果您之前手动调整过系数值，重算将应用最新的系数配置。"
    *   **操作**: `[取消]`, `[开始重算]`。

#### B. 导入数据弹窗设计 (保持 v2.0)
*   **Tabs**: 电表 / 计量点。
*   **上传**: 拖拽批量上传。
*   **反馈**: 结果统计 Chips + 错误列表 + 智能缺失预警。

### 4.2 Workbench 交互更新

#### 操作区 (Action Area)
*   **[ 🔄 单户增量聚合 ]**:
    *   主推操作。快速把刚才补录的文件算进去。
*   **[ ⚙️ 单户重算... ]**:
    *   次要操作（链接/小按钮）。
    *   点击弹出与 Dashboard 类似的 **重算弹窗**，但范围 **锁定为当前客户**。
    *   常用于：由于之前系数配错了，现在想把去年的数据按新系数重刷一遍。

## 5. UI 视觉规范

*   **状态色谱**:
    *   **Pending (蓝 #1976d2)**: 已上传但未聚合。
    *   **Health (绿 #2e7d32)**: 数据完整且准确。
    *   **Warning (橙 #ed6c02)**: 可靠率不足（缺MP）或 误差轻微（1-3%）。
    *   **Critical (红 #d32f2f)**: 完全断点 或 误差严重（>3%）。
    *   **Neutral (灰 #9e9e9e)**: 无数据周期或未签约时段。

*   **响应式适配**:
    *   **Desktop**: 完整 Split Button 交互。
    *   **Mobile**: 聚合按钮简化为 `[聚合]`，点击弹出 ActionSheet 选择 `[增量]` 或 `[重算]`。
