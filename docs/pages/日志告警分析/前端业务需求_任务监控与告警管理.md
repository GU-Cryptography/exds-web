> **文档版本**: v1.0

> **目标读者**: 前端开发团队  
> **后端项目**: EXDS-STEF (电力市场短期能源预测系统)

---

## 一、业务背景

### 1.1 系统概述

EXDS-STEF 项目运行着一个统一任务调度服务,负责执行各类预测任务、数据下载任务等。该服务通过 MongoDB 记录任务执行日志、系统告警,并支持通过 MongoDB 接收远程命令触发任务。

**前后端关系**:
- 前后端项目**共用同一个 MongoDB 数据库** (`exds`)
- 后端任务服务**写入**日志和告警数据
- 前端 Web 应用**读取并展示**这些数据
- 前端可通过**写入命令集合**触发后端任务执行

### 1.2 核心功能需求

前端需要实现以下功能:

1. **任务监控面板** - 查看任务执行历史、状态、耗时等
2. **告警中心** - 查看和管理系统告警,支持确认、解决、忽略操作
3. **远程任务触发** - 手动触发预测任务、查看命令执行状态

## 二、数据库集合详解

### 2.1 任务执行日志 (`task_execution_logs`)

#### 用途
记录所有任务的完整生命周期,包括开始时间、结束时间、状态、结果详情等。

#### Schema 定义

```javascript
{
  "_id": ObjectId("..."),
  
  // ========== 核心标识 ==========
  "task_id": "forecast_d1_price_20260203_092000_a1b2",  // 唯一任务ID
  "start_time": ISODate("2026-02-03T09:20:00Z"),        // 任务开始时间
  "end_time": ISODate("2026-02-03T09:21:15Z"),          // 任务结束时间
  "duration": 75.3,                                      // 耗时(秒)
  
  // ========== 分类字段 ==========
  "service_type": "forecast",     // 服务大类: forecast, download, compute
  "task_type": "d1_price",        // 具体任务类型
  "task_name": "D-1 日前价格预测", // 任务显示名称
  
  // ========== 状态与摘要 ==========
  "status": "SUCCESS",            // 状态: RUNNING, SUCCESS, FAILED, WAITING, PARTIAL
  "summary": "D-1价格预测完成: 2026-02-04, 96点, 均价415.23元",  // 预测任务
  // "summary": "D-1 模型周训练完成: WMAPE准确率 92.50%, R² 0.9100, 版本 5",  // 训练任务
  
  // ========== 触发信息 ==========
  "trigger_type": "schedule",     // 触发方式: schedule, manual, event
  
  // ========== 可扩展详情 (按任务类型差异化) ==========
  "details": {
    // 预测任务详情示例
    "target_date": "2026-02-04",
    "forecast_id": "d1_20260204_092000",
    "point_count": 96,
    "price_stats": {
      "mean": 415.23,
      "min": 218.50,
      "max": 602.10,
      "std": 45.67
    },
    "model_path": "models_saved/d1_forecast/d1_price_model.joblib",
    "confidence_intervals": {
      "ci_80": true,
      "ci_90": true
    }
    
    // 训练任务详情示例
    // "metrics": {
    //   "WMAPE_Accuracy": 92.5,
    //   "R2": 0.91,
    //   "MAE": 18.5
    // },
    // "model_version": 5,
    // "promoted_to_production": true
    
    // 下载任务详情示例
    // "collection": "weather_forecasts",
    // "record_count": 168,
    // "date_range": { "start": "2026-02-03", "end": "2026-02-09" }
  },
  
  // ========== 错误信息 (失败时填充) ==========
  "error": {
    "code": null,           // 错误代码 (如 "MODEL_LOAD_ERROR")
    "message": null,        // 错误信息
    "stack_trace": null     // 堆栈跟踪 (调试用)
  }
}
```

#### 状态说明

| 状态 | 说明 | 前端展示建议 |
|-----|------|-------------|
| `RUNNING` | 任务正在执行 | 蓝色进度条 + "执行中" |
| `SUCCESS` | 任务成功完成 | 绿色勾 + "成功" |
| `FAILED` | 任务执行失败 | 红色叉 + "失败" |
| `WAITING` | 等待数据 | 黄色警告 + "等待数据" |
| `PARTIAL` | 部分完成 | 橙色 + "部分完成" |

#### 数据产生机制

**自动产生** (由任务服务自动写入):
- 所有定时调度任务 (每日 06:00 天气下载、09:20 预测等)
- 所有远程命令触发的任务

**写入时机**:
1. 任务开始 → 写入 `status: RUNNING`
2. 任务结束 → 更新 `status: SUCCESS/FAILED`, 填充 `end_time`, `duration`, `details`

#### 索引

```javascript
db.task_execution_logs.createIndex({ "task_id": 1 }, { unique: true })
db.task_execution_logs.createIndex({ "start_time": -1 })
db.task_execution_logs.createIndex({ "service_type": 1, "task_type": 1, "start_time": -1 })
db.task_execution_logs.createIndex({ "status": 1, "start_time": -1 })
```

### 2.2 系统告警 (`system_alerts`)

#### 用途
记录需要人工关注的异常事件,如任务失败、数据缺失、系统错误等。

#### Schema 定义

```javascript
{
  "_id": ObjectId("..."),
  
  // ========== 核心标识 ==========
  "alert_id": "alert_20260203_093500_a1b2c3",  // 唯一告警ID
  "created_at": ISODate("2026-02-03T09:35:00Z"),
  
  // ========== 告警级别与分类 ==========
  "level": "P1",                    // 级别: P0 (紧急), P1 (重要), P2 (普通)
  "category": "TASK_FAILED",        // 类别: 见下表
  
  // ========== 告警内容 ==========
  "title": "D-1 日前价格预测 执行失败",
  "content": "模型文件不存在: models_saved/d1_forecast/d1_price_model.joblib",
  
  // ========== 状态管理 ==========
  "status": "ACTIVE",               // 状态: ACTIVE, RESOLVED, IGNORED
  
  // ========== 关联信息 ==========
  "service_type": "forecast",       // 关联的服务类型
  "task_type": "d1_price",          // 关联的任务类型
  "related_task_id": "forecast_d1_price_20260203_092000_a1b2",  // 关联的任务ID
  
  // ========== 可扩展上下文 (按告警类型差异化) ==========
  "context": {
    // 数据缺失告警示例
    // "target_date": "2026-02-04",
    // "missing_items": ["daily_release"],
    // "expected_count": 96,
    // "actual_count": 0
    
    // 任务失败告警示例
    "error_code": "MODEL_LOAD_ERROR",
    "model_path": "models_saved/d1_forecast/d1_price_model.joblib"
  },
  
  // ========== 解决跟踪 ==========
  "resolved_at": null,              // 解决时间
  "resolved_by": null,              // 解决人
  "resolution_note": null           // 解决备注
}
```

#### 告警级别说明

| 级别 | 说明 | 前端展示建议 | 通知方式 |
|-----|------|-------------|---------|
| `P0` | 紧急阻断 | 红色 + 闪烁 | 数据库 + 邮件 |
| `P1` | 重要失败 | 橙色 + 高亮 | 数据库 + 邮件 |
| `P2` | 一般警告 | 黄色 | 仅数据库 |

#### 告警类别说明

| 类别 | 说明 | 常见场景 |
|-----|------|---------|
| `DATA_MISSING` | 数据缺失 | 预测前检查发现数据未到位 |
| `TASK_FAILED` | 任务失败 | 任务执行过程中抛出异常 |
| `TASK_TIMEOUT` | 任务超时 | 任务执行时间超过预期 |
| `LOGIN_FAILED` | 登录失败 | RPA 登录交易中心失败 |
| `SYSTEM_ERROR` | 系统错误 | 数据库连接失败、服务崩溃 |
| `QUALITY_ISSUE` | 质量问题 | 预测结果异常、准确度过低 |

#### 数据产生机制

**自动产生**:
- 任务执行失败 → 自动创建 `TASK_FAILED` 告警 (P1)
- 数据检查失败 → 自动创建 `DATA_MISSING` 告警 (P1)

**自动消除**:
- 当同类型任务重新执行成功时,相关的 `ACTIVE` 告警会自动标记为 `RESOLVED`

#### 索引

```javascript
db.system_alerts.createIndex({ "alert_id": 1 }, { unique: true })
db.system_alerts.createIndex({ "created_at": -1 })
db.system_alerts.createIndex({ "status": 1, "level": 1, "created_at": -1 })
db.system_alerts.createIndex({ "service_type": 1, "task_type": 1, "status": 1 })
db.system_alerts.createIndex({ "related_task_id": 1 })
```

### 2.3 任务命令队列 (`task_commands`)

#### 用途
前端通过写入此集合向后端任务服务发送命令,后端轮询此集合并执行命令。

#### Schema 定义

```javascript
{
  "_id": ObjectId("..."),
  
  // ========== 核心标识 ==========
  "command_id": "cmd_20260203_100000_abc",  // 唯一命令ID: cmd_{timestamp}_{uuid}
  "command": "run",                          // 命令类型: run, cancel
  
  // ========== 任务分类 ==========
  "service_type": "forecast",               // 服务大类: forecast, download, compute
  "task_type": "d1_price",                  // 具体任务类型
  
  // ========== 任务参数 ==========
  "parameters": {
    "target_date": ISODate("2026-02-04"),   // 目标日期 (预测任务)
    "force": false                           // 是否强制重新执行
    // ... 其他任务特定参数
  },
  
  // ========== 状态跟踪 ==========
  "status": "pending",                      // 状态: pending, running, completed, failed
  "priority": 5,                            // 优先级 1-10 (1最高, 默认5)
  
  // ========== 元数据 ==========
  "created_by": "admin@example.com",        // 创建人
  "created_at": ISODate("2026-02-03T10:00:00Z"),
  "started_at": null,                       // 开始执行时间
  "completed_at": null,                     // 完成时间
  "result_message": null,                   // 结果消息
  "error_message": null                     // 错误消息
}
```

#### 命令状态说明

| 状态 | 说明 | 前端展示建议 |
|-----|------|-------------|
| `pending` | 等待执行 | 灰色 + "排队中" |
| `running` | 正在执行 | 蓝色 + "执行中" |
| `completed` | 执行完成 | 绿色 + "已完成" |
| `failed` | 执行失败 | 红色 + "失败" |

#### 数据生命周期

```
1. 前端写入 → status: pending
2. 后端轮询 (每 120 秒) → 发现 pending 命令
3. 后端更新 → status: running, started_at: 当前时间
4. 后端执行任务
5. 后端更新 → status: completed/failed, completed_at: 当前时间
6. 7天后自动删除 (TTL 索引)
```

#### 支持的任务类型

| service_type | task_type | 说明 | 必需参数 |
|-------------|-----------|------|---------|
| `forecast` | `d1_price` | D-1 日前价格预测 | `target_date` |
| `forecast` | `d2_price` | D-2 日前价格预测 | `target_date` |
| `forecast` | `d1_train` | D-1 模型训练 | 无 |
| `download` | `weather_download` | 天气数据下载 | 无 |
| `download` | `fuel_download` | 燃料价格下载 | 无 |

#### 索引

```javascript
db.task_commands.createIndex({ "command_id": 1 }, { unique: true })
db.task_commands.createIndex({ "status": 1, "priority": 1, "created_at": 1 })
db.task_commands.createIndex({ "service_type": 1, "task_type": 1, "status": 1 })
db.task_commands.createIndex({ "created_at": 1 }, { expireAfterSeconds: 604800 })  // 7天自动过期
```

## 三、推荐 API 接口设计

### 3.1 任务执行日志 API

#### 3.1.1 获取任务列表

```http
GET /api/v1/tasks/execution-logs
```

```typescript
{
  page?: number;           // 页码 (默认 1)
  page_size?: number;      // 每页数量 (默认 20)
  service_type?: string;   // 筛选服务类型
  task_type?: string;      // 筛选任务类型
  status?: string;         // 筛选状态
  start_date?: string;     // 筛选开始日期 (YYYY-MM-DD)
  end_date?: string;       // 筛选结束日期 (YYYY-MM-DD)
}
```

**响应示例**:
```json
{
  "success": true,
  "data": {
    "total": 150,
    "page": 1,
    "page_size": 20,
    "items": [

}
```

## 四、前端页面功能设计

### 4.1 任务监控面板

#### 页面布局

```
┌─────────────────────────────────────────────────────┐
│  任务监控                                            │
├─────────────────────────────────────────────────────┤
│  [统计卡片区]                                        │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐              │
│  │今日  │ │成功  │ │失败  │ │运行中│              │
│  │任务  │ │任务  │ │任务  │ │任务  │              │
│  └──────┘ └──────┘ └──────┘ └──────┘              │
├─────────────────────────────────────────────────────┤
│  [筛选器]                                            │
│  服务类型: [全部▼] 任务类型: [全部▼] 状态: [全部▼] │
│  日期范围: [2026-02-01] ~ [2026-02-03]              │
├─────────────────────────────────────────────────────┤
│  [任务列表表格]                                      │
│  任务名称 | 状态 | 开始时间 | 耗时 | 操作            │
│  ─────────────────────────────────────────────────  │
│  D-1价格预测 | ✓成功 | 09:20 | 75s | [详情]       │
│  天气下载    | ✓成功 | 06:00 | 12s | [详情]       │
│  D-1价格预测 | ✗失败 | 09:20 | 5s  | [详情][重试] │
└─────────────────────────────────────────────────────┘
```

#### 功能要点

1. **实时刷新**: 每 30 秒自动刷新任务列表
2. **状态图标**: 
   - 成功: 绿色勾 ✓
   - 失败: 红色叉 ✗
   - 运行中: 蓝色旋转图标 ⟳
   - 等待: 黄色警告 ⚠
3. **详情弹窗**: 点击"详情"显示完整的 `details` 和 `error` 信息
4. **重试按钮**: 失败任务可点击"重试",调用触发任务 API

#### 功能要点

1. **级别颜色**:
   - P0: 红色 🔴

   - P2: 黄色 🟡

   - **详情**: 显示完整的 `content` 和 `context`
   - **解决**: 弹出对话框,填写解决备注,调用解决 API
   - **忽略**: 弹出对话框,填写忽略原因,调用忽略 API
3. **关联跳转**: 点击告警可跳转到关联的任务详情 (通过 `related_task_id`)

## 五、业务规则与注意事项

### 5.1 数据刷新策略

| 页面 | 刷新频率 | 说明 |
|-----|---------|------|
| 任务监控 | 30 秒 | 自动刷新任务列表 |
| 告警中心 | 60 秒 | 自动刷新告警列表 |
| 命令历史 | 10 秒 | 仅刷新 `pending` 和 `running` 状态的命令 |

### 5.2 权限控制

| 操作 | 建议权限 |
|-----|---------|
| 查看任务日志 | 所有用户 |
| 查看告警 | 所有用户 |
| 解决/忽略告警 | 管理员 |
| 触发任务 | 管理员 |

### 5.3 时区处理

- 数据库存储: UTC 时间 (`ISODate`)
- 前端显示: 转换为本地时区 (Asia/Shanghai)

### 5.4 分页建议

- 默认每页 20 条
- 最大每页 100 条
- 提供"加载更多"或传统分页组件

### 5.5 错误处理

**后端任务服务离线**:
- 检测方式: 查询最近 5 分钟是否有新任务日志
- 前端提示: "任务服务可能离线,请联系管理员"

**命令长时间 pending**:
- 检测方式: 命令创建超过 5 分钟仍为 `pending`
- 前端提示: "命令执行延迟,任务服务可能繁忙"

## 六、技术建议

### 6.1 前端技术栈

- **框架**: React / Vue
- **UI 组件库**: Ant Design / Material-UI
- **状态管理**: Redux / Vuex

3. **操作反馈**: 解决告警、触发任务后显示成功提示
