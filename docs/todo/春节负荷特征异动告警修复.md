# 节假日负荷特征异动告警修复方案 (基于 holiday_service)

## 1. 问题背景分析
用户反馈在春节放假期间、以及节假日**前后（减产/复工期）**，系统在进行“客户负荷特征分析”时，产生了大量的异常告警（如：力度异动-骤降、剧烈异动，以及伴随负荷大幅降低产生的**形状异动**和**重心异动**）。
**根本原因**：目前的异常检测规则只基于数据的统计学规律判定，缺乏业务上的长假上下文。工厂在假日前后停工/复产导致的负荷突变、曲线形状改变、高峰时段转移，在假期上下文中均属于“合理预期”，不应触发告警。

## 2. 详细改造方案

### 2.1 基于 holiday_service 增加“假期影响期”判断
**文件**: `webapp/services/characteristics/rules/calendar.py`
系统中已存在 `holiday_service.py`，我们将基于它来推导当前日期是否处于“假期影响期”。特别是春节，其影响范围远大于法定假期。

```python
from datetime import datetime, timedelta
from typing import Optional

def get_holiday_impact_info(dt: datetime) -> Optional[str]:
    """
    基于 holiday_service 判断给定日期是否在假期或其周边影响期内
    """
    from webapp.services.holiday_service import get_holiday_service
    svc = get_holiday_service()
    
    # 1. 是否正好是法定节假日当天
    holiday_name = svc.get_holiday_name(dt.date())
    if holiday_name:
        return f"{holiday_name}假期"
        
    # 2. 扩大范围扫描（春节影响期：假日前7天 到 假日后10天）
    # 向前扫描（查找dt是否在节后）
    for i in range(1, 11): 
        if svc.get_holiday_name(dt.date() - timedelta(days=i)) == '春节':
            return "春节后复工期"
    # 向后扫描（查找dt是否在节前）
    for i in range(1, 8):
        if svc.get_holiday_name(dt.date() + timedelta(days=i)) == '春节':
            return "春节前减产期"
            
    # 3. 其他长假扩大扫描（国庆、五一：前2天 到 后3天）
    for i in range(1, 4):
        h_name = svc.get_holiday_name(dt.date() - timedelta(days=i))
        if h_name in ['国庆节', '劳动节']:
            return f"{h_name}后复工期"
    for i in range(1, 3):
        h_name = svc.get_holiday_name(dt.date() + timedelta(days=i))
        if h_name in ['国庆节', '劳动节']:
            return f"{h_name}前减产期"
            
    return None
```

### 2.2 在上下文 LabelingContext 注入假期信息
**文件**: `webapp/services/characteristics/engine/context.py` 和 `webapp/services/characteristics/service.py`
在 `LabelingContext` 此时原本就有一个 `holiday_info` 字段。
在 `service.py` 构建上下文时赋予具体的影响期状态：
```python
from .rules.calendar import get_holiday_impact_info

holiday_info = get_holiday_impact_info(date)

context = LabelingContext(
    # ...
    holiday_info=holiday_info
)
```

### 2.3 短期异动规则的“假期定向免疫”机制
**文件**: `webapp/services/characteristics/rules/anomaly.py`
为所有的核心短期异动规则按方向加入保护逻辑：

**1. 完全豁免（假期期间忽略的指标）：**
由于停产和只保留保安电源，这些指标必然变动，没有任何告警价值。
- `ShapeAnomalyRule` (形状异动)
- `PeakShiftAnomalyRule` (重心异动)
- `VolatilityAnomalyRule` (波动/规律异动)
- `AnomalyDetectionRule` (IForest兜底)

**拦截判定逻辑**:
```python
if context.holiday_info:
    return None
```

**2. 单向豁免（仅保留正向激增的指标）：**
为了捕捉客户未报备的**提前复工**或**意外非计划用电**，保留激增方向的规则，但屏蔽骤降方向。

- `ScaleAnomalyRule` (力度 - 停产/骤降)
修改逻辑：如果 `context.holiday_info` 存在：
  - 不再报“疑似停产” ( ratio < 0.20 )
  - 不再报“骤降” ( deviation < -0.50 )
  - **保留**“激增” ( deviation > 0.50 )的告警

- `ExtremeAnomalyRule` (剧烈异动)
修改逻辑：如果 `context.holiday_info` 存在：
  - 不再报“负荷率剧变”
  - 不再报“电量异常低” ( z_score > 2.5 但方向向下 )
  - **保留**“电量异常高” ( z_score > 2.5 且方向向上 )的告警。可以在原因中附加说明：“(发生于{context.holiday_info}期间，疑似违规用电/提前复产)”

- `DaySurgeRule` (日环比突变)
修改逻辑：如果 `context.holiday_info` 存在：
  - 不再报“骤降” ( change_ratio < -1.0 )
  - **保留**“激增” ( change_ratio > 1.0 )的告警。

这样不仅消除了烦人的“合法减产”噪音，还强化了对于“计划外复工”的风险监控能力。

## 3. 验证计划
修改完代码后，针对一个此前在春节前减产期、或春节假期间报过“形状异动”/“力度异动”的客户，通过手动执行该日期的特征分析，确认不再生成任何相关告警标签。
