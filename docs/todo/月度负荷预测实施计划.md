# 客户负荷预测服务实现方案 (月度交易版)

## 目标描述
构建**月度及多月交易辅助预测**功能。
系统支持在每月交易窗口期（通常为26日前），对所有活跃客户进行**未来3个月**的日度负荷预测，并聚合为月度电量，指导交易决策。

**核心特性**：
1.  **一次性快照**：每个交易周期（月）仅保留一份最终的预测结果（"交易定稿"），用于后续考核与复盘。
2.  **长周期覆盖**：单次预测覆盖未来 3 个完整的自然月（例如2月做预测，覆盖3月、4月、5月）。
3.  **灵活修正**：支持按“单客户”或“全系统”进行修正（整体修正需分摊到户）。
4.  **复盘能力**：提供历史预测版本与实际结算电量的比对分析。

## 用户审查事项
> [!IMPORTANT]
> **业务规则**：
> - **预测周期 (`forecast_cycle`)**：以**下个自然月**作为周期标识。
>     - 例：1月20日执行预测，目标是 2/3/4月，则 `forecast_cycle = "2026-02"`。
> - **锁定机制**：
>     - 在每月交易截止日（如26日）前，可多次重算 or 修正。
>     - 截止后，建议锁定该周期的预测结果（或逻辑上不再引用），作为历史归档。
> - **整体修正逻辑**：
>     - 如果用户在总览界面输入“总电量 +5%”，系统将自动按比例分摊更新到所有客户的日预测值中。

## 3. 核心算法与策略 (v3)

### 3.1 客户分层策略 (Pareto Principle)
为了平衡计算精度与效率，系统将客户分为两类：
1.  **重点客户 (Major Clients)**：按历史电量排序，累计占比达到 **80%** 的前 N 位客户。
    -   **策略**：执行精细化的**单户日度预测**（考虑排班、天气、趋势）。
2.  **中小客户 (Minor Clients)**：剩余的 20% 客户。
    -   **策略**：不进行单户预测。基于**重点客户的预测总量**和**历史占比**进行整体推算。
    -   **占比计算区间**：从 **上月1日** 至 **预测执行日前一日**。
        -   例如：2月20日执行预测，统计区间为 1月1日 ~ 2月19日。
    -   公式：$P_{minor\_total} = P_{major\_total} \times \frac{\sum H_{minor}}{\sum H_{major}}$ (区间内实测电量之和)。

### 3.2 重点客户日预测算法

$$
P_{day} = (L_{base\_type} \times K_{weather} \times K_{trend}) + \Delta_{adj}
$$

其中所有 $K$ 系数均为**算法自动计算**，无需人工初始输入。

#### A. 分时基准值 ($L_{base\_type}$)
不再使用单一的日均值，而是根据客户标签（`周末双休` / `周末单休` / `全年无休`）分别统计：
-   **$L_{work}$ (工作日基准)**：取近30天内**工作日**的日均电量。
-   **$L_{rest}$ (休息日基准)**：取近30天内**休息日**（周末/节假日）的日均电量。
-   **$Curve_{typical}$ (典型曲线)**：取 `short_term.avg_curve` (48点归一化曲线)。

#### B. 气象/季节补偿 ($K_{weather}$)
针对 `气温敏感` (Summer/Winter) 客户，算法自动计算补偿系数：
-   **计算逻辑**：
    1.  获取该客户**去年同季节月份**（如去年7月）的日均电量 $L_{last\_season}$．
    2.  获取该客户**去年普通月份**（如去年5月）的日均电量 $L_{last\_normal}$．
    3.  计算增长比率 $\alpha = L_{last\_season} / L_{last\_normal}$．
    4.  **$K_{weather} = \alpha$**。

#### C. 趋势系数 ($K_{trend}$)
-   自动引用特征画像中的 `trend_slope` (年化斜率)。
-   计算公式：$K_{trend} = 1 + (\text{trend_slope} \times \frac{\text{预测日} - \text{特征日}}{365})$。

### 3.3 预测流程总结
1.  **筛选**：识别重点客户。
2.  **单户计算**：
    -   计算日总电量：$P_{total} = (L \times K \times \dots) + \Delta$。
    -   **时段分解**：$P_{48} = P_{total} \times Curve_{typical}$ (将日电量按典型曲线拆解为48点)。
3.  **整体推算**：
    -   先算出重点客户的总量 $Sum_{major} = \sum P_{total}$。
    -   总预测 $Total = Sum_{major} \times (1 + Ratio_{minor/major})$。

## 4. 前端页面规划建议 (增强交易视图)

页面标题：**月度交易负荷预测工作台**

### 4.1 顶部控制栏
-   **周期选择器**：下拉选择预测起始月（默认下月，如 "2026-03"）。
-   **全局指标卡**：
    -   预测总电量 (kWh)
    -   重点客户数 (电量占比 >80%)
    -   **峰谷比/负荷率**：反映整体负荷特性的关键指标。
-   **导出报表**：一键导出符合交易中心要求的**48时段电量申报表** (Excel)。

### 4.2 Tab 1: 交易总览 (Portfolio View)
-   **月度同比分析图**：展示去年、今年实测与未来3个月预测的月电量对比（虚线框显示预测）。
-   **聚合48时段负荷曲线 (核心)**：
    -   展示全量客户聚合后的**典型日负荷曲线**（48点）。
    -   **交互**：支持切换查看 M1, M2, M3 的聚合曲线。
    -   **用途**：交易员根据此曲线进行分时段交易头寸计算。
-   **整体修正面板**：输入系统级增减百分比，实时更新聚合曲线。

### 4.3 Tab 2: 客户清单与时段修正 (Customer Details)
-   **客户清单表格**：
    -   列：客户、标签、预测总电量、**峰电量/谷电量**、修正系数、操作。
-   **单户详情看板 (Modal)**：
    -   **时段分配图**：展示该客户 48 点负荷分配。
    -   **日历修正**：
        -   在日历上标记特需日期。
        -   **典型曲线切换**：若客户近期生产模式变更（如由三班倒改为两班倒），支持快速切换/重选基准典型曲线。

### 4.4 Tab 3: 复盘与时段精度 (Accuracy Analysis)
-   **准确性看板**：
    -   **总体偏差**：月度总电量误差率。
    -   **时段偏差**：分析哪些时段（如午峰、晚峰）预测最不准，辅助调整气象补偿逻辑。
-   **历史版本比对**：查看过去几个交易周期定稿后的表现。


### 数据集结构调整 (v3.1)

#### `customer_trading_forecasts`
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `_id` | ObjectId | 唯一ID |
| `customer_id` | String | S客户ID |
| `forecast_cycle` | String | 周期标识 (YYYY-MM) |
| `months` | Object | **月度聚合数据 (关键)** <br> Map: `year_month` -> `MonthlyDetail` |
| `daily_data` | Array | 日详情列表 (90天) |

#### 新增 `MonthlyDetail` 结构
用于支持月度分时交易：
```json
{
  "total_energy": 150000.5,       // 月度总电量 (kWh)
  "monthly_profile": [            // 月度48时段聚合曲线 (该月所有天对应时段之和)
    3000.1, 3200.5, ..., 2800.0   // [00:00, 00:30, ... 23:30]
  ],
  "stats": {
    "tip": 20000,   // 尖峰总电量
    "peak": 50000,  // 高峰总电量
    "flat": 40000,  // 平段总电量
    "valley": 40000 // 低谷总电量
  }
}
```

#### `daily_data` 元素结构
```json
{
  "date": "2026-03-01",
  "value": 5000.0,
  "profile": [100.0, 105.0, ...], // 日48点曲线
  "is_adjusted": false
}
```


## 拟定修改

### 后端开发

#### 1. [NEW] `webapp/services/forecast/`
新建 `TradingForecastService` 模块。

-   **`models.py`**:
    -   `TradingForecastRequest`: `{ target_month: "2026-03", adjustments: ... }`
-   **`service.py`**:
    -   `generate_cycle_forecast(target_month)`:
        -   **步骤 1**: 确定时间范围（`target_month` 起始日 ~ `target_month + 2` 结束日，共约90天）。
        -   **步骤 2**: 获取该期间所有活跃客户。
        -   **步骤 3**: 逐户计算日度预测（算法同前：基准+日历+趋势+气象敏感性）。
        -   **步骤 4**: 存入/更新 `customer_trading_forecasts` 集合。
    -   `apply_correction(cycle, customer_id, ratio/value)`: 单户修正。
    -   `apply_global_correction(cycle, ratio)`: 整体修正（批量更新数据库）。
    -   `get_forecast_accuracy(customer_id, cycle)`: 计算偏差。

#### 2. [MODIFY] `webapp/api/v1_router.py`
-   注册 `/trading-forecast` 路由。
    -   `POST /generate`: 触发/重算某周期的预测。
    -   `POST /correct`: 提交修正。
    -   `GET /dashboard`: 获取某周期的汇总数据（按月聚合）。
    -   `GET /check`: 获取历史预测与实际值的对比数据。

### 数据集结构设计 (v3)

#### `customer_trading_forecasts` - 客户交易预测表
存储每个客户在每个交易周期的预测详情。

-   **主键逻辑**: 每个客户每个交易周期一条记录。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `_id` | ObjectId | 唯一ID |
| `customer_id` | String | S客户ID |
| `forecast_cycle` | String | **交易周期标识** (YYYY-MM)，如 "2026-03" |
| `execution_date` | String | 实际执行日期 (YYYY-MM-DD) |
| `is_locked` | Boolean | 是否已锁定/归档 |
| `months` | Object | 月度汇总 (冗余查询用) `{ "2026-03": 1000, "2026-04": 1200 ... }` |
| `daily_data` | Array | 90天日详情 |
| `daily_data.$.date` | String | 日期 |
| `daily_data.$.value` | Number | 预测电量 |
| `daily_data.$.is_adjusted`| Boolean| 是否有人工修正 |

-   **索引**: `(customer_id: 1, forecast_cycle: 1)` [Unique]
